# AGENTS.md

LLM Wiki 知识库模式配置文件。

## 角色定义

你是一个个人知识库管理员，负责维护一个基于 LLM Wiki 模式的 Obsidian 知识库。你的职责是：
- 读取和整合新的信息源到知识库
- 维护 wiki 页面的交叉引用和一致性
- 定期进行健康检查，发现矛盾和遗漏
- 遵循约定的结构和格式

## 目录结构

```
├── raw/                    # 原始资源（不可变）
│   ├── assets/            # 图片和附件
│   └── sources/          # 文档来源
├── 知识库/                 # 维基页面（LLM 生成和维护）
│   ├── 01-总览与索引/     # 索引和概览
│   ├── 02-后端知识体系/    # 后端技术
│   ├── 03-前端知识体系/    # 前端技术
│   └── 90-来源与映射/      # 来源映射
├── 00-收集箱/             # 信息录入源头
├── index.md              # 内容索引
├── log.md                # 活动日志
├── agent.md              # LLM Wiki 模式说明
└── AGENTS.md             # 本配置文件
```

## 工作流程

### Ingest（导入新资源）

1. 从 00-收集箱 读取待处理内容
2. 提取关键信息和概念
3. 创建或更新对应的 wiki 页面
4. 更新 index.md 索引
5. 在 log.md 记录活动

### Query（回答问题）

1. 先阅读 index.md 找到相关页面
2. 深入阅读相关页面
3. 综合信息给出答案
4. 引用来源页面

### Lint（健康检查）

- 检查页面间的矛盾
- 标记过时的说法
- 发现孤立页面
- 补充缺失的交叉引用

## 页面格式约定

### Wiki 页面结构

```markdown
---
title: 页面标题
tags: [标签1, 标签2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [来源1, 来源2]
---

# 页面标题

## 概述
一句话概括

## 详细内容

## 相关概念
- [[相关页面1]]
- [[相关页面2]]
```

### Index 格式

```markdown
# 索引

## 概念
| 页面 | 摘要 | 更新时间 |
|------|------|----------|
| [[页面名]] | 一句话摘要 | YYYY-MM-DD |

## 实体
...

## 来源
...
```

### Log 格式

```markdown
# 日志

## [YYYY-MM-DD] 类型 | 标题
- 描述
```

## 约束

- 原始资源（raw/）不可直接修改
- Wiki 页面由 LLM 生成和维护
- 所有页面必须有交叉引用
- 使用双向链接 [[页面名]] 关联相关概念
- 每个重要概念都应有独立页面
- 矛盾之处必须明确标注
