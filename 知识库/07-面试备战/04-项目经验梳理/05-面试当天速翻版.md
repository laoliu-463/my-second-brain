# 面试当天速翻版

> 面试当天快速过一遍用。每条：问法 → 顺口答（一句话）→ 追问关键词。看完直接开口讲。

## 一、自我介绍（1分钟）

**问法**：请做一下自我介绍。

**顺口答**：面试官好，我叫曹家宁，大连交通大学软件工程本科二年级，应聘 Java 后端实习。学过 Java 基础、Spring Boot、MySQL、Redis，熟悉微服务组件 Nacos/Gateway/OpenFeign/Sentinel/Seata。做过两个项目：AI 面试助手（RAG+向量数据库+异步任务）和教务管理系统（Spring Cloud 微服务+Redis+分布式事务）。希望实习参与真实业务提升后端能力。

## 二、AI 面试助手项目（5-8分钟）

### 1. 项目整体

**问法**：AI 面试助手是做什么的？为什么做？

**顺口答**：基于 RAG 的简历分析和面试辅助系统。用户上传简历 → 切 chunk → Embedding 向量化 → 存 pgvector。用户提问 → 检索最相关 chunk → 拼 Prompt → 大模型生成回答。参考了开源 RAG 思路，但文本切分、向量检索参数、Redis Stream 异步任务和结构化输出都是我独立实现的。

**追问关键词**：RAG 原理 / pgvector 选型 / 独立实现了哪些部分

---

### 2. RAG 原理

**问法**：RAG 是什么？怎么工作的？

**顺口答**：Retrieval-Augmented Generation，先检索相关内容再生成回答。用户提问 → 问题向量化 → 检索 topK 最相似 chunk → 拼 Prompt → 大模型回答。保证回答有依据，降低 token 消耗。

**追问关键词**：余弦相似度 / HNSW 算法 / topK 控制

---

### 3. pgvector 选型

**问法**：为什么用 pgvector 不用 Milvus 或 Chroma？

**顺口答**：pgvector 是 PostgreSQL 扩展，项目已有 PG 启用就行，无需额外部署。Milvus 要单独运维，Chroma 是 Python 库不兼容 Java。数据量中等，pgvector 毫秒级检索完全够用。

**追问关键词**：轻量落地 / HNSW 近似最近邻

---

### 4. 文本切分策略

**问法**：为什么要切分 chunk？不直接整篇向量化？

**顺口答**：太长粒度粗，检索不准。500 字符切一个 chunk，邻 chunk 重叠 50 字符，保证语义不被切断，检索精度大幅提升。

**追问关键词**：重叠切分 / topK 从 3 扩到 5

---

### 5. Redis Stream 异步

**问法**：Redis Stream 在项目里干什么？为什么不用 MQ？

**顺口答**：上传后解析+向量化+生成报告耗时，扔 Redis Stream 异步跑，接口秒级返回任务 ID，轮询查进度。项目已有 Redis，轻量够用，不需要引入 RabbitMQ。

**追问关键词**：消费者组 / PENDING 列表 / 消费者挂了怎么办

---

### 6. AI 回答跑偏

**问法**：AI 乱说怎么办？

**顺口答**：三层控制：检索层过滤相关度低的 chunk；Prompt 里写"只能根据提供内容回答，不知道就说不知道"；输出校验 JSON 格式完整性，异常重试最多 3 次。

**追问关键词**：检索层 / Prompt 指令约束 / 结构化校验

---

### 7. 项目难点

**问法**：项目最难的问题是什么？

**顺口答**：两个——检索不准和接口慢。检索不准：改重叠切分+扩 topK 到 5。接口慢：Redis Stream 异步化，响应从 20 秒降到 200 毫秒。排查用 Arthas trace 定位瓶颈。

**追问关键词**：Arthas trace / chunk 调优 / 异步化

---

### 8. 真实性和上线

**问法**：项目上线了吗？照教程做的？

**顺口答**：个人作品集没大规模用户，但工程完整：接口分层、异常处理、异步任务、Docker 部署。参考开源 RAG 思路，异步任务和结构化输出独立实现。

---

### 9. 检索精度怎么调

**问法**：RAG 检索有时候不准，怎么优化？

**顺口答**：chunk 切小一点，按段落或固定长度切；topK 只拿最相关片段；Prompt 里只拼这些片段，减少无关上下文干扰。

**追问关键词**：chunk 策略 / topK / 上下文噪音控制

---

### 10. 上传慢怎么处理

**问法**：上传简历后处理很慢，怎么解决？

**顺口答**：解析、切分、向量化、生成报告都放 Redis Stream 异步跑，接口先返回任务 ID。消费者组处理任务，失败还能重试。

**追问关键词**：Redis Stream / 消费者组 / pending / 重试

---

### 11. AI 乱说怎么办

**问法**：AI 回答不准确怎么控制？

**顺口答**：三层控制：检索层只给相关 chunk；Prompt 里写“只能根据提供内容回答，不知道就说不知道”；输出按优势、风险点、改进建议结构化返回。

**追问关键词**：Prompt 约束 / 结构化输出 / 降低幻觉

---

### 12. 和开源项目区别

**问法**：和 Snailclimb/interview-guide 有什么区别？

**顺口答**：参考了 RAG 架构思路，但 chunk 切分、topK 检索、Redis Stream 异步任务和结构化输出是我自己按简历场景改造实现的。

**追问关键词**：参考思路 / 独立改造 / 场景化优化

---

### 13. 工程化怎么样

**问法**：项目上线了吗？可扩展性如何？

**顺口答**：个人作品集，没有大规模用户，但接口分层、异常处理、异步任务、Docker 部署都按真实工程方式做了，后续可以直接继续扩展。

**追问关键词**：工程化 / Docker / 可扩展性

## 三、教务管理系统项目（5-8分钟）

### 1. 项目整体

**问法**：教务管理系统做什么？你负责什么？

**顺口答**：微服务教务平台，学生/课程/课时管理。我负责后端全部接口、数据库设计、微服务集成、Redis 缓存和 Seata 分布式事务。

**追问关键词**：为什么微服务 / 模块拆分逻辑

---

### 2. 为什么微服务

**问法**：为什么要拆成微服务？单体不行吗？

**顺口答**：模块边界清晰，独立开发部署降低耦合。想通过项目练习 Spring Cloud Alibaba 全家桶。小规模可以单体，但我主要练微服务治理和分布式能力。

**追问关键词**：Nacos / Gateway / OpenFeign / Sentinel / Seata

---

### 3. Sentinel 怎么配

**问法**：你项目里 Sentinel 怎么做限流？

**顺口答**：关键接口配 QPS 限流，比如课程查询、消课接口；热点课程 ID 再配热点参数限流。超过阈值就降级，返回兜底数据或者提示稍后重试。

**追问关键词**：QPS 规则 / 热点参数 / 动态规则热加载

---

### 4. 服务失败怎么办

**问法**：服务间调用失败或超时怎么处理？

**顺口答**：OpenFeign 调用，Sentinel 做熔断和限流；非核心数据走降级返回缓存或默认值，耗时操作改 Redis Stream 异步处理，保证主链路可用。

**追问关键词**：降级策略 / 异步补偿 / 可用性优先

---

### 5. 跨服务事务

**问法**：报名课程时创建订单和扣课时，怎么保证一致性？

**顺口答**：Seata AT 模式做全局事务，两个服务放在同一个事务链路里，任意一个失败就整体回滚，保证要么都成功要么都失败。

**追问关键词**：AT 模式 / 全局事务 / 回滚日志

---

### 6. SQL 优化

**问法**：消课统计接口慢，怎么优化？

**顺口答**：先补学生 ID、课程 ID、日期这些高频字段索引，再用 Redis 缓存统计结果，然后看 EXPLAIN 优化多表关联，避免全表扫描。

**追问关键词**：索引 / Redis / EXPLAIN / 多表查询

---

### 7. 组件职责

**问法**：Nacos、Gateway、OpenFeign、Sentinel、Seata 各自干什么？

**顺口答**：Nacos = 注册中心+配置中心；Gateway = 统一网关（路由+鉴权+限流）；OpenFeign = 声明式 HTTP 调用；Sentinel = 流量控制+熔断降级；Seata = 分布式事务保证跨服务一致性。

**追问关键词**：Nacos 注册发现 / Sentinel 多维度限流 / Seata AT 模式原理

---

### 8. Seata 分布式事务

**问法**：Seata 是怎么工作的？

**顺口答**：AT 模式，执行 SQL 前拍照，执行后比对数据快照，有变化自动生成逆向 SQL 回滚。跨服务方法加 @GlobalTransactional 注解即可，业务代码零侵入。

**追问关键词**：undo log / 全局锁 / 拍照比对

---

### 9. Seata 难点

**问法**：Seata 遇到过什么坑？

**顺口答**：全局锁超时。查 seata.grouplist 配置、调事务超时时间（30s→60s）、优化 TC 网络通信才解决。

**追问关键词**：全局锁超时排查 / 超时参数调优

---

### 10. 接口慢怎么优化

**问法**：接口响应慢怎么排查解决？

**顺口答**：Arthas trace 定位。三个方向：高频字段加索引；多个 Feign 调用用 CompletableFuture 并行；热点数据 Redis 缓存。实测 800ms→300ms。

**追问关键词**：Arthas trace / CompletableFuture 并行 / 热点缓存

---

### 11. Redis 用法

**问法**：Redis 在教务系统怎么用？

**顺口答**：两个用法——缓存学生/课程信息减少查库；Redis Stream 做异步任务队列削峰。

**追问关键词**：String/Hash 缓存 / Stream 消费者组

---

### 12. 项目难点

**问法**：项目最难的部分是什么？

**顺口答**：Seata 配置调优最难。全局锁超时、undo log 记录、seata.grouplist 配置，调了很久才摸清楚。接口优化第二，用 Arthas 定位 N+1 问题，批量查询替代逐条查。

**追问关键词**：Seata 超时 / N+1 批量治理

## 四、Java 基础（5分钟速过）

| 问题 | 一句话顺口答 |
|---|---|
| ArrayList vs LinkedList | ArrayList 查询 O(1) 但插入要搬移元素；LinkedList 插入 O(1) 但查询要遍历 |
| HashMap 底层 | JDK 1.8 数组+链表+红黑树，链表超 8 转树，查询 O(log n) |
| ConcurrentHashMap | JDK 1.8 用 CAS+synchronized 锁头节点，读无锁写加锁，并发度更高 |
| synchronized vs ReentrantLock | synchronized 是 JVM 关键字自动释放；ReentrantLock 可重入、可打断、可设公平锁 |
| 线程池工作流程 | 任务来 → 核心线程执行 → 队列排队 → 最大线程执行 → 拒绝策略 |
| CountDownLatch vs CyclicBarrier | Latch 倒数计数等所有人完成；Barrier 等所有人到齐再一起走 |
| ThreadLocal 坑 | 用完必须 remove，否则内存泄漏；线程池复用线程会读脏数据 |

## 五、数据库（3分钟速过）

| 问题 | 一句话顺口答 |
|---|---|
| B+ 树优势 | 多路平衡树，层级少 IO 少；叶子节点链表支持范围查询 |
| 索引失效场景 | 索引列用函数、类型不匹配、LIKE 前缀%、OR 一边无索引、NOT IN |
| 事务隔离级别 | 读未提交/读已提交/可重复读/串行化；MySQL 默认可重复读 |
| MVCC 原理 | undo log 版本链 + trx_id 快照读；读已提交每次生成新快照，可重复读事务开始生成 |
| SQL 优化 | EXPLAIN 分析 + 建覆盖索引 + 避免 SELECT * + 分页 LIMIT + 批量操作 |
