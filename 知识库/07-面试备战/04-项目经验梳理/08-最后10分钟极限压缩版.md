# 最后10分钟极限压缩版

## 概述

这页不是完整复习资料，而是面试开始前最后 10 分钟用的极限压缩版。只保留最容易开口卡住、但又最容易被问到的 20 个点。目标不是“全”，而是“稳”，看完就能张口。

## 详细内容

## 用法

- 第 1 遍：只看粗体问法，脑子里先回想答案
- 第 2 遍：看顺口答，把开头两句背稳
- 第 3 遍：只扫“追问词”，防止第二轮追问断掉

---

## 1. 自我介绍

**问法**：请做一下自我介绍。

**顺口答**：面试官您好，我叫曹家宁，大连交通大学软件工程本科二年级，求职方向是 Java 后端实习。我主要用 Spring Boot、MySQL、Redis，也接触了 Spring Cloud Alibaba 这套微服务组件。做过两个项目，一个是 AI 面试助手，核心是 RAG、Redis Stream 和 Prompt 控制；另一个是教务管理系统，核心是微服务、Seata 分布式事务、Redis 缓存和 Sentinel 限流。我希望通过实习把这些技术真正放到业务开发里。

**追问词**：技术栈 / 两个项目 / 为什么想做后端

---

## 2. AI 项目整体

**问法**：AI 面试助手是做什么的？

**顺口答**：这是一个基于 RAG 的简历分析和面试辅助系统。用户上传简历后，后端先解析文本、切 chunk、做向量化并存到 pgvector；用户提问时，再检索最相关的简历片段，拼 Prompt 给大模型生成回答。

**追问词**：RAG / pgvector / 独立实现了什么

---

## 3. 为什么做 AI 项目

**问法**：你为什么做这个项目？

**顺口答**：因为我不想只做一个简单调模型 API 的 demo，我想把 RAG、向量检索、异步任务和输出控制串成一条完整后端链路。这个项目更能体现我把 AI 想法工程化落地的能力。

**追问词**：工程化 / 不是纯 demo / 体现什么能力

---

## 4. RAG 是什么

**问法**：RAG 是什么？怎么工作的？

**顺口答**：RAG 就是检索增强生成，先从知识库里检索相关内容，再把检索结果和问题一起交给模型生成回答。这样做的好处是回答更有依据，也能减少无关上下文带来的干扰。

**追问词**：topK / 余弦相似度 / 为什么比直接喂全文更好

---

## 5. pgvector 选型

**问法**：为什么用 pgvector，不用 Milvus？

**顺口答**：因为项目本身已经有 PostgreSQL，pgvector 直接启用扩展就能用，接入成本低，数据量中等时性能也够。Milvus 功能更强，但要单独部署维护，对我这个 Java 项目来说成本更高。

**追问词**：轻量落地 / 运维成本 / 数据量

---

## 6. 文本切分

**问法**：为什么要切 chunk？

**顺口答**：如果整篇简历只做一个向量，粒度太粗，检索很容易不准。我会按固定长度加少量重叠去切 chunk，让每个向量更聚焦一段语义，这样检索精度更高。

**追问词**：重叠切分 / 语义完整性 / topK

---

## 7. Redis Stream

**问法**：Redis Stream 在 AI 项目里干什么？

**顺口答**：简历解析、切分、向量化、报告生成都比较耗时，我把它们放到 Redis Stream 里异步执行，接口先快速返回任务 ID。这样用户不用同步等完整条链路跑完。

**追问词**：消费者组 / pending / 为什么不用 RabbitMQ

---

## 8. AI 回答不准怎么办

**问法**：如果 AI 回答跑偏了，你怎么控制？

**顺口答**：我会从三层控制。第一层是检索层，只给模型最相关的 chunk；第二层是 Prompt 层，明确要求只能基于给定内容回答；第三层是输出层，要求结构化返回，异常时可以校验和重试。

**追问词**：Prompt 约束 / 结构化输出 / 降低幻觉

---

## 9. 教务系统整体

**问法**：教务管理系统做什么？你负责什么？

**顺口答**：这是一个面向教培场景的微服务教务系统，主要有学生、课程、课时这些模块。我主要负责后端接口、数据库设计、微服务组件集成，以及 Redis 缓存和分布式事务这块。

**追问词**：为什么微服务 / 你参与深度 / 哪块最核心

---

## 10. 为什么微服务

**问法**：单体不也能做吗？为什么拆微服务？

**顺口答**：小项目当然也能用单体，但这个项目里学生、课程、课时这些模块边界比较清晰，拆开后更方便独立开发和扩展。对我来说，更重要的是通过它练到服务治理、远程调用和分布式事务这些能力。

**追问词**：Nacos / Gateway / OpenFeign / Seata

---

## 11. 微服务组件职责

**问法**：Nacos、Gateway、Feign、Sentinel、Seata 各干什么？

**顺口答**：Nacos 做注册发现和配置中心，Gateway 做统一入口和路由鉴权，OpenFeign 做服务间调用，Sentinel 做限流和熔断，Seata 做分布式事务保证跨服务一致性。

**追问词**：最难的是哪个 / 为什么网关放入口 / Seata 原理

---

## 12. Seata 一致性

**问法**：报名课程同时创建订单和扣课时，怎么保证一致性？

**顺口答**：我用 Seata AT 模式把多个服务操作放进同一个全局事务里。只要有一个子服务失败，整个事务就会回滚，保证要么都成功，要么都失败。

**追问词**：undo log / 全局锁 / 为什么不用 TCC

---

## 13. Sentinel 限流

**问法**：Sentinel 在项目里怎么用？

**顺口答**：我会给关键接口做 QPS 限流，比如课程查询、消课接口；如果某个课程 ID 特别热，还会配热点参数限流。超过阈值后做降级，返回缓存数据或者提示稍后重试。

**追问词**：100 QPS 怎么定 / 热点参数 / 动态热加载

---

## 14. 接口慢怎么办

**问法**：接口慢你怎么排查？

**顺口答**：我会先定位瓶颈在哪层，比如数据库、远程调用还是本地计算。排查时会先看慢 SQL 和索引，再看 Feign 调用是不是串行太多，最后考虑 Redis 缓存和异步化。实际项目里我用 Arthas trace 辅助定位过耗时点。

**追问词**：Arthas / EXPLAIN / CompletableFuture 并行

---

## 15. 线程池

**问法**：线程池核心参数和执行流程是什么？

**顺口答**：最核心的是核心线程数、最大线程数、阻塞队列和拒绝策略。执行流程是任务先进核心线程，核心满了进队列，队列满了再扩到非核心线程，还满就触发拒绝策略。

**追问词**：CallerRunsPolicy / 为什么不用 Executors / 线程命名

---

## 16. HashMap 和 ConcurrentHashMap

**问法**：HashMap 和 ConcurrentHashMap 有什么区别？

**顺口答**：HashMap 线程不安全，多线程下可能出现数据覆盖甚至异常；ConcurrentHashMap 是线程安全的，JDK 1.8 里主要通过 CAS 加局部锁来提升并发性能。

**追问词**：CAS / 局部锁 / 为什么不用 Hashtable

---

## 17. ACID 和 MVCC

**问法**：事务 ACID 和 MVCC 怎么结合项目讲？

**顺口答**：单服务内部写操作我会用 Spring 事务保证 ACID 里的原子性和一致性；并发查询场景下，MVCC 让读写尽量不互相阻塞，提升查询性能；跨服务时再用 Seata 把多个本地事务串成一个全局事务。

**追问词**：隔离级别 / 快照读 / 当前读 / 全局事务

---

## 18. Redis 三大问题

**问法**：缓存穿透、击穿、雪崩怎么答？

**顺口答**：穿透是查不存在的数据，常用空值缓存或布隆过滤器；击穿是热点 key 失效瞬间大量并发回源，常用互斥锁或逻辑过期；雪崩是大量 key 同时过期，常用 TTL 随机化和高可用部署。

**追问词**：热点 key / 逻辑过期 / 为什么能保护数据库

---

## 19. Spring IoC / AOP

**问法**：Spring IoC 和 AOP 怎么简短解释？

**顺口答**：IoC 就是把对象创建和依赖管理交给 Spring；AOP 本质是代理模式，把日志、事务、权限这些横切逻辑从业务代码里抽出去统一处理。

**追问词**：JDK 动态代理 / CGLIB / 事务为什么基于 AOP

---

## 20. 反问环节

**问法**：你有什么问题想问我的吗？

**顺口答**：我一般会问两个。第一，团队目前主要技术栈和中间件是什么，我可以提前熟悉；第二，实习生有没有 mentor 带、有没有机会参与核心业务开发。这两个问题既自然，也能体现我更关注成长和业务。

**追问词**：技术栈 / mentor / 核心业务

---

## 最后 1 分钟提醒

- 主线别乱：自我介绍 -> AI 项目 -> 教务系统 -> Java / 数据库 / Redis -> 反问
- 被打断很正常，先接追问，再回主线
- 开头两句最重要，先把每题的第一句说稳
- 不会就收口：先讲自己项目里实际怎么做，再补理论
