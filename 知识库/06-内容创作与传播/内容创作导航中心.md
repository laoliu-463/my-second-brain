## 公众号发布（核心工具：md2wechat）

**工具：** `md2wechat` — 用 Markdown 写公众号，一键转微信草稿

**CLI 安装命令（任选一种）：**

```bash
# Homebrew（mac）
brew install geekjourneyx/tap/md2wechat

# npm
npm install -g @geekjourneyx/md2wechat

# Go
go install github.com/geekjourneyx/md2wechat-skill/cmd/md2wechat@v2.0.7
```

**核心工作流：**

```bash
# 1. 检查最终 metadata 和发布风险
md2wechat inspect article.md

# 2. 生成本地 HTML 预览
md2wechat preview article.md

# 3. 转换并发送到微信草稿箱
md2wechat convert article.md --draft --cover cover.jpg
```

**写作风格辅助：**

```bash
# 用 Dan Koe 风格帮你写文章
md2wechat write --style dan-koe

# 生成匹配的封面提示词
md2wechat write --style dan-koe --cover
```

**md2wechat 项目路径：** `/mnt/d/Projects/OpenSource/md2wechat-skill/`

## 小红书发布

**发布稿归档路径：** `/mnt/d/小红书发布稿/`

**文件规范：**

```
E01_发布稿_标题.md       ← Markdown 正文
E01_封面图_标题.png      ← 封面图（3:4 竖版）
E01_行动清单_标题.png    ← 可选行动清单图
```

**发布流程：**
1. 打开 `小红书笔记模板.md`
2. 填写 7 段式内容
3. 制作封面图（3:4 竖版，突出反常识结论）
4. 发布到小红书
5. 记录数据反馈到追踪表

**模板路径：** `/mnt/d/小红书发布稿/小红书笔记模板.md`

## 播客生产闭环

**完整工作流：**

```
选题库 → Sources原资料 → NotebookLM → 认知笔记/播客音频
                                         ↓
                              小宇宙发布 ← 播客文稿
                                         ↓
                              小红书引流（不发外链）
```

**每周节奏：**

| 周一 | 周二 | 周三 | 周四 | 周五 |
|------|------|------|------|------|
| 选题+收集 | 笔记+音频 | 文稿+小宇宙 | 小红书发布 | 复盘数据 |

**关键文件路径：**

| 文件 | 路径 |
|------|------|
| 工作流说明 | `/mnt/d/工作流程说明.md` |
| 选题追踪表 | `/mnt/d/选题库/选题追踪表.md` |
| 认知笔记模板 | `/mnt/d/NotebookLM 输出/认知笔记模板.md` |
| 播客文稿模板 | `/mnt/d/NotebookLM 输出/播客文稿模板.md` |
| 小红书模板 | `/mnt/d/小红书发布稿/小红书笔记模板.md` |

## 选题库

**路径：** `/mnt/d/选题库/选题追踪表.md`

**选题标准：**
- 有明确痛点场景
- 有反常识结论
- 有可执行行动
- 目标听众能产生"必须听完整版"的冲动

**状态流转：** 待选 → 资料收集中 → NotebookLM处理中 → 文稿完成 → 已发布
