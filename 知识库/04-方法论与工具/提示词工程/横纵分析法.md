# 横纵分析法

## 概述

横纵分析法是一种用于深度研究分析的提示词方法论，通过**纵向（时间维度）** 和**横向（空间维度）** 两个维度，对研究对象进行完整、深入的分析。

## 变量定义

```
研究对象 = [待填写]
```

> 使用时只需修改「研究对象」的定义，其余内容保持不变。

## 一、纵向分析（Diachronic / Longitudinal）

沿时间轴，完整还原「研究对象」从诞生到现在的发展全貌。

### 1. 起源追溯

- 它诞生的背景是什么？
- 基于什么技术/理念/需求而来？
- 创始团队或核心推动者是谁？
- 当时的行业环境是什么样的？

### 2. 诞生节点

- 明确的首次发布/成立/提出时间
- 最初的形态和定位

### 3. 演进历程

从诞生到现在，按时间顺序梳理所有关键节点：

- 重大版本更新
- 融资事件
- 团队变动
- 战略转型
- 技术架构变化
- 用户规模里程碑
- 重大合作或收购
- 公关危机或争议事件

### 4. 决策逻辑

在每个关键节点上，尽可能还原决策背后的原因：

- 为什么选了 A 而不是 B？
- 当时面对的约束条件是什么？

### 5. 叙事要求

不要写成干巴巴的年表。要用故事的方式把发展史串起来，让读者能感受到因果关系和时代脉络。越详细、越多元越好，把相关的人物、事件、背景信息都拽进来。

## 二、横向分析（Synchronic / Cross-sectional）

以当前时间点为切面，将「研究对象」与同赛道的竞品/同类进行全面对比。

### 竞品情况判断

根据竞品数量选择对应场景：

| 场景 | 条件 | 分析重点 |
|------|------|----------|
| **场景A** | 无直接竞品 | 分析：为何没有竞品？替代方案？潜在竞争者？ |
| **场景B** | 1-2个竞品 | 逐一深入对比 |
| **场景C** | 3个及以上竞品 | 选取3-5个代表性竞品详细分析 |

### 对比维度

1. **核心差异对比**：
   - 技术路线/核心方法论/底层逻辑
   - 产品形态/商业模式/组织结构
   - 目标用户/受众/适用场景
   - 核心优势与明显短板
   - 定价策略/资源投入/规模体量

2. **用户视角**：
   - 每个竞品的真实用户口碑
   - 社区评价中的优点和槽点
   - 用户实际使用方式和官方定位的偏差

3. **生态位分析**：
   - 在整个赛道版图中占据什么位置
   - 填补了什么空白
   - 是否在跟谁正面竞争

4. **趋势判断**：
   - 竞争格局中的走向
   - 机会和风险各是什么

## 三、写作风格要求

这不是一份冷冰冰的咨询报告，而是一篇让人能从头读到尾的深度研究。

| 要求 | 说明 |
|------|------|
| **可读性优先** | 写得像优质深度报道，有节奏感和画面感 |
| **叙事驱动** | 纵向部分要有故事弧线，有起承转合 |
| **观点有据** | 每个观点必须有事实支撑，推测要明确标注 |
| **用人话写** | 避免空洞形容词，用具体细节代替概括性陈述 |
| **对比有温度** | 讲清楚每个竞品"活成了什么样" |

### 禁止使用的词汇

- "赋能"
- "抓手"
- "打造闭环"
- 其他咨询公司式套话

## 四、篇幅要求

| 部分 | 字数范围 | 说明 |
|------|----------|------|
| **纵向分析** | 6000-15000字 | 报告主体，历史越长越靠近上限 |
| **横向分析** | 3000-10000字 | 竞品越多篇幅越长 |
| **横纵交汇总结** | 1500-3000字 | 精华段，给出综合判断 |
| **全文总计** | 10000-30000字 | 写到该停的地方自然停 |

### 竞品场景对应篇幅

- **场景A（无竞品）**：横向分析3000字左右，重点在替代方案和潜在竞争者
- **场景B（1-2个）**：每个竞品详细展开
- **场景C（3个及以上）**：每个主要竞品至少1500字

## 五、输出格式要求

1. 先输出纵向分析（发展史叙事），再输出横向分析（竞品对比）
2. 纵向部分以时间叙事为主线，避免纯粹列表格式
3. 横向部分可适当使用对比表格辅助，核心分析必须是文字论述
4. 报告最后加「横纵交汇」总结：结合纵向发展脉络和横向竞争格局，给出对「研究对象」当前位置和未来走向的判断
5. 所有信息尽可能标注来源或时间节点
6. 无法确认的信息标注为推测，不要编造

## 适用范围

此分析法适用于以下类型的研究对象：

| 类型 | 示例 |
|------|------|
| **产品/工具** | Hermes Agent、Cursor、Claude Code |
| **公司/组织** | Anthropic、字节跳动、OpenAI |
| **技术概念** | MCP协议、RAG、Agent框架 |
| **人物** | 行业关键人物的职业轨迹 |
