# Java 方法重载与重写

> 一句话记住：重载看参数，重写看继承。

---

## 一、核心区别总览

| 对比点 | 重载 Overload | 重写 Override |
|--------|-------------|-------------|
| 发生位置 | 同一个类中 | 父子类之间 |
| 方法名 | 必须相同 | 必须相同 |
| 参数列表 | **必须不同** | **必须相同** |
| 返回值 | 无关 | 相同或是父类返回值的子类 |
| 访问权限 | 无关 | 不能比父类更小 |
| 异常 | 无关 | 不能抛更大的检查异常 |
| 多态类型 | 编译期多态 | 运行期多态 |
| 典型场景 | 同一功能多种参数形式 | 子类改写父类行为 |

---

## 二、方法重载 Overload

### 定义

同一个类中，方法名相同，**参数列表不同**（个数、类型、顺序）。

```java
public int add(int a, int b) { return a + b; }
public double add(double a, double b) { return a + b; }  // 参数类型不同
public int add(int a, int b, int c) { return a + b + c; } // 参数个数不同
```

### 不构成重载的情况

```java
// 参数名不同不算重载（类型和个数相同）
public void test(String a) {}
public void test(String name) {}  // ✗ 编译报错

// 返回值不同不算重载
public int add(int a, int b) { return a + b; }
public double add(int a, int b) { return a + b; }  // ✗ 编译报错

// 访问修饰符不同不算重载
public void test(int a) {}
private void test(int a) {}  // ✗ 编译报错
```

## 三、方法重写 Override

### 定义

子类继承父类后，对父类已有方法重新实现。要求：方法名相同、参数列表相同、返回值类型相同或是父类返回值的子类。

### @Override 注解

告诉编译器检查是否真正重写了父类方法，写错方法名会报错。**推荐重写时必加。**

### 重写规则

| 规则 | 说明 |
|------|------|
| 方法名+参数 | 必须完全相同 |
| 返回值 | 相同或是父类返回值类型的子类（协变返回） |
| 访问权限 | 不能比父类更小（`protected` → `public` ✓） |
| 异常 | 不能抛出比父类更宽的检查异常 |
| `private` 方法 | 不能被重写（子类不可见） |
| `static` 方法 | 不能重写（属于类，重复定义叫隐藏） |
| `final` 方法 | 不能重写 |

### 不能重写的示例

```java
class Parent {
    private void test() {}   // 子类不可见，重写不了
    static void staticMethod() {}  // static 不涉及多态
    public final void finalMethod() {}  // final 禁止重写
}
```

## 四、面试标准回答

**Q：重载和重写的区别？**

> 重载发生在同一个类中，方法名相同但参数列表不同（个数、类型、顺序），与返回值和访问修饰符无关，属于编译期多态，编译器根据参数确定调用哪个方法。重写发生在父子类之间，子类对父类已有方法重新实现，要求方法名和参数列表相同、返回值类型相同或是父类返回值的子类、访问权限不能比父类小、不能抛比父类更宽的检查异常，属于运行期多态。

## 五、测试答案

**题1**：重载（参数类型不同：`int` vs `String`）

**题2**：重写（父子类间、方法名和参数相同）

**题3**：不能编译。重载判断不看返回值，只看方法名和参数列表，两个 `add(int, int)` 签名完全相同，冲突。

**题4**：不是重写，是重载。`test(String)` 和 `test()` 参数列表不同，属于同一个类（从重写角度看父子的 test 是不同方法）。

**题5**：不能编译。`final` 方法不能被重写。
