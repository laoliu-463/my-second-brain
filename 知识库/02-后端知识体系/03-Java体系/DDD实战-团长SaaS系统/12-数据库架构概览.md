# 12-数据库架构概览

## 1. 概述

团长 SaaS 使用 **PostgreSQL** 作为主数据库（Docker Compose 命名卷持久化），使用 **MyBatis-Plus** 作为 ORM 层。数据库采用**手动管理**模式（无 Flyway/Liquibase），所有变更通过 `src/main/resources/db/` 目录下的 `.sql` 脚本管理。

## 2. 数据库变更管理

### 2.1 变更脚本目录

```
src/main/resources/db/
├── alter-cso-sample-attribution-indexes.sql
├── alter-menu-permission-model.sql
├── alter-merchant-owner-columns.sql
├── alter-merchant-ownership.sql
├── alter-order-attribution-mvp-v2.sql
├── alter-order-attribution-mvp.sql
├── alter-order-pick-source-length.sql
├── alter-order-talent-id.sql
├── alter-partitioned-order-log-pk-default.sql
├── alter-pick-source-mapping-colonel-buyin-id.sql
├── alter-product-following.sql
├── alter-product-main-chain.sql
└── ... (共 27 个 alter 脚本)
```

共 **27 个 ALTER 脚本**，均为追加性变更（加字段、加索引），无回滚脚本。

### 2.2 变更节奏

根据脚本名称判断变更内容：

| 变更类型 | 典型脚本 | 说明 |
|---|---|---|
| 订单归因 | `alter-order-attribution-mvp*.sql` | MVP 阶段的归因字段扩展 |
| PickSource | `alter-order-pick-source-length.sql` | pick_source 字段长度调整 |
| 样品归属 | `alter-cso-sample-attribution-indexes.sql` | 样品订单归因索引 |
| 达人字段 | `alter-order-talent-id.sql` | 订单表加 talent_id 字段 |
| 商家归属 | `alter-merchant-owner*.sql` | merchant 表归属字段变更 |
| PickSourceMapping | `alter-pick-source-mapping-colonel-buyin-id.sql` | 归因映射表加 colonel_buyin_id |

### 2.3 字段命名规范

数据库使用**下划线命名**（PostgreSQL 标准）：

| Java 字段 | DB 字段 |
|---|---|
| `douyinUid` | `douyin_uid` |
| `colonelBuyinId` | `colonel_buyin_id` |
| `pickSource` | `pick_source` |
| `serviceFeeContribution` | `service_fee_contribution` |

MyBatis-Plus 通过 `@TableField("xxx")` 注解映射。

## 4. 索引设计

### 4.1 归因相关索引

```sql
-- alter-cso-sample-attribution-indexes.sql
-- colonelsettlement_order 表的归因字段组合索引
CREATE INDEX idx_cso_talent_uid ON colonelsettlement_order(extra_data->>'talent_uid');
CREATE INDEX idx_cso_author_id ON colonelsettlement_order(extra_data->>'author_id');
CREATE INDEX idx_cso_activity_id ON colonelsettlement_order(activity_id);
CREATE INDEX idx_cso_channel_user_id ON colonelsettlement_order(channel_user_id);
```

### 4.2 PickSourceMapping 索引

```sql
-- alter-pick-source-mapping-colonel-buyin-id.sql
-- 用于归因反查：pick_source → colonelBuyinId
CREATE INDEX idx_psm_short_id ON pick_source_mapping(short_id);
CREATE INDEX idx_psm_colonel_buyin_id ON pick_source_mapping(colonel_buyin_id);
CREATE INDEX idx_psm_activity_product ON pick_source_mapping(activity_id, product_id);
```

## 5. JSONB 字段使用

`colonelsettlement_order.extra_data` 和 `colonel_activity.extra_data` 使用 PostgreSQL JSONB 类型存储抖店原始响应：

```sql
-- 归因时从 extra_data 取达人 UID（优先级：talent_uid > author_id > talent_name）
COALESCE(extra_data->>'talent_uid', extra_data->>'author_id', talent_name)
```

## 6. 金额存储

所有金额字段使用 **Long 类型（单位：分）**存储，不使用 DECIMAL：

| 字段 | 说明 |
|---|---|
| `order_amount` | 订单金额（分） |
| `settle_colonel_commission` | 团长佣金（分） |
| `settle_colonel_tech_service_fee` | 技术服务费（分） |
| `settle_second_colonel_commission` | 二级团长佣金（分） |

避免浮点精度问题，计算时在应用层转换。

## 7. 软删除

所有表使用 `deleted` 字段（Integer）标记软删除，MyBatis-Plus 查询自动拼接 `.eq(deleted, 0)`：

```java
// BaseEntity 统一提供
wrapper.eq("deleted", 0);
```
