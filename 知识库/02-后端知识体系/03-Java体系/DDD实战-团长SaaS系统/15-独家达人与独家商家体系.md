# 15-独家达人与独家商家体系

## 1. 概述

独家体系是团长 SaaS 的**归因优先策略**：当达人或商家的服务费贡献达到阈值时，自动将其纳入独家资源池，**强制归因**给对应的团长或商家，确保高价值客户不被其他团队撬动。

核心机制：每月按上一自然月的服务费贡献进行评估，达到阈值则本月该达人/商家强制归属对应团长。

## 2. 独家达人（ExclusiveTalent）

### 2.1 数据模型（exclusive_talent）

```java
// ExclusiveTalent.java
public class ExclusiveTalent {
    String talentUid;         // 达人抖音 UID
    UUID talentId;           // 达人系统 ID
    UUID userId;             // 认领人（团长用户）
    UUID deptId;             // 部门
    Integer exclusiveType;   // 独家类型（1=普通独家）
    String effectiveMonth;    // 生效月份（yyyy-MM 格式，如 "2026-05"）
    Integer status;          // 状态（1=生效）
    Integer deleted;         // 软删除
    BigDecimal ratio;        // 服务费占比（%）
    Integer sampleCount;     // 样品数量
}
```

### 2.2 独家生效规则（双条件同时满足）

每月 1 日（或手动触发）评估上一自然月数据：

```
条件 1：服务费占比 ≥ 阈值（默认 70%）
        该达人对某团长的服务费 / 该达人总服务费 ≥ 70%

条件 2：有效样品数 ≥ 阈值（默认 10 个）
        上月该团长与该达人的有效样品申请数 ≥ 10

同时满足 → 该达人本月独家归属该团长
```

```java
// ExclusiveTalentService.java:78
if (ratio.compareTo(ratioThreshold) >= 0 && sampleCount >= sampleThreshold) {
    upsertExclusive(row, totalFee, ratio, sampleCount, applyMonth);
}
```

阈值配置：

| 配置 Key | 默认值 | 说明 |
|---|---|---|
| `talent.exclusive.service_fee_ratio` | 70（%） | 服务费占比门槛 |
| `talent.exclusive.monthly_samples` | 10（个） | 有效样品数门槛 |

### 2.3 服务费占比计算

```java
// ExclusiveTalentService.java:74-76
BigDecimal ratio = BigDecimal.valueOf(row.channelFee())
        .multiply(BigDecimal.valueOf(100))
        .divide(BigDecimal.valueOf(totalFee), 2, RoundingMode.HALF_UP);
// ratio = 该团长为该达人带来的服务费 / 该达人总服务费 × 100，保留两位小数
```

### 2.4 查询当前生效独家

```java
// ExclusiveTalentService.java:88-104
public AttributionService.ExclusiveOwner findActiveOwnerByTalentUid(String talentUid) {
    String month = YearMonth.now().format(MONTH_FORMATTER);  // 当前月份
    ExclusiveTalent match = exclusiveTalentMapper.selectOne(
            .eq(ExclusiveTalent::getTalentUid, talentUid)
            .eq(ExclusiveTalent::getEffectiveMonth, month)
            .eq(ExclusiveTalent::getStatus, 1)
            .eq(ExclusiveTalent::getDeleted, 0)
            .last("limit 1"));
    if (match == null) return null;
    return new AttributionService.ExclusiveOwner(match.getUserId(), match.getDeptId());
}
```

## 3. 独家商家（ExclusiveMerchant）

### 3.1 数据模型（exclusive_merchant）

```java
// ExclusiveMerchant.java
public class ExclusiveMerchant {
    String merchantId;        // 商家 ID
    UUID userId;             // 归属团长
    UUID deptId;             // 部门
    Integer exclusiveType;    // 独家类型
    String effectiveMonth;    // 生效月份（yyyy-MM）
    Integer status;          // 状态（1=生效）
    Integer deleted;         // 软删除
}
```

### 3.2 独家生效规则（单条件）

```java
// ExclusiveMerchantService.java:44-61
// 条件：服务费占比 ≥ 阈值（默认 70%）
// 该商家给某团长带来的服务费 / 该商家总服务费 ≥ 70%
if (ratio.compareTo(threshold) >= 0) {
    upsertMerchantExclusive(row, totalFee, ratio, applyMonth);
}
```

阈值配置：

| 配置 Key | 默认值 |
|---|---|
| `merchant.exclusive.service_fee_ratio` | 70（%） |

### 3.3 与独家达人的区别

| 维度 | 独家达人 | 独家商家 |
|---|---|---|
| 条件数量 | 双条件（服务费占比 + 样品数） | 单条件（服务费占比） |
| 样品门槛 | 有（≥10 个/月） | 无 |
| 配置 Key 前缀 | `talent.exclusive` | `merchant.exclusive` |

## 4. 独家体系在归因中的优先级

### 4.1 归因决策树中的位置

在 `AttributionService.resolveAttribution()` 中，独家判断位于**通道一（pick_source）和通道二（colonel_order_info）之前**：

```
AttributionService.resolveAttribution(order, source)
       ↓
① 独家商家检查（ExclusiveMerchantService）
       ↓ 有独家商家 → 直接归因给对应团长，跳过后续通道
       ↓ 无独家商家
② 独家达人检查（ExclusiveTalentService）
       ↓ 有独家达人 → 直接归因给对应团长，跳过后续通道
       ↓ 无独家达人
③ pick_source 通道（外部推广链接）
④ colonel_order_info 通道（抖店原生归因）
```

### 4.2 独家优先的原因

独家体系是**商家/团长高意愿绑定**，服务费贡献达标说明该团长对商家/达人关系深，优先级高于链路归因。

## 5. 月度评估触发

| 服务 | 触发方式 |
|---|---|
| 独家达人评估 | 手动或定时 `evaluatePreviousMonthAndApplyCurrentMonth()` |
| 独家商家评估 | 手动或定时 `evaluatePreviousMonthAndApplyCurrentMonth()` |

均使用 `@Transactional`，确保评估和写入在同一事务中。
