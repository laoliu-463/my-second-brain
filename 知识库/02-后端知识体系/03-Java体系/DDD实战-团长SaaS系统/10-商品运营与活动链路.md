# 10-商品运营与活动链路

## 1. 概述

团长 SaaS 的商品运营链路是从抖店同步商品 → 创建活动 → 达人对接 → 生成推广链接 → 产生订单的全流程。本章重点覆盖商品运营状态机、活动与商品的关系、以及推广链接生成逻辑。

## 2. 核心实体

### 2.1 商品快照（ProductSnapshot）

每次抖店商品同步时，商品基础信息存入 `product_snapshot` 表（只追加不修改，作为历史快照）：

| 字段 | 说明 |
|---|---|
| `product_id` | 抖店商品 ID |
| `activity_id` | 关联的活动 ID |
| `title` | 商品标题 |
| `image_url` | 商品主图 |
| `price` | 价格 |
| `commission_rate` | 佣金率 |
| `status` | 抖店侧状态 |
| `detail_url` | 商品详情页 URL |
| `sync_time` | 同步时间 |

### 2.2 商品运营状态（ProductOperationState）

商品在特定活动下的运营状态（一个商品在多个活动下有多条记录）：

```java
// ProductOperationState.java
public class ProductOperationState {
    String activityId;           // 活动 ID
    String productId;           // 抖店商品 ID
    String boundActivityId;     // 绑定的活动 ID（可能与 activityId 不同）
    UUID assigneeId;            // 负责人（团长用户）
    String bizStatus;           // 业务状态（PENDING_REVIEW/ASSIGNED/LINKED 等）
    Integer auditStatus;        // 审核状态
    String promoteLink;         // 推广链接
    String shortLink;           // 短链接
    Integer promotionScene;     // 推广场景
    Boolean selectedToLibrary; // 是否入选选品库
    LocalDateTime selectedAt;   // 入选选品库时间
}
```

每次生成推广链接时存入 `promotion_link` 表：

## 3. 商品运营状态机（ProductBizStatus）

### 3.1 状态定义

```
PENDING_REVIEW  →  待审核（商品入库后默认状态）
REJECTED       →  已拒绝
ASSIGNED       →  已分配（已指定负责人）
PENDING_LINK   →  待转链
LINKED         →  已转链（已生成推广链接）
ARCHIVED       →  已归档
```

### 3.2 状态流转

```
                  ┌──────────────────────────────────┐
                  │                                  ▼
           PENDING_REVIEW ──→ REJECTED
                  │
                  ▼
            ASSIGNED ──→ PENDING_LINK ──→ LINKED
                  │                           │
                  │                           ▼
                  │                       ARCHIVED
                  │
                  └─────────────────────────────┘
                            （可回到 PENDING_LINK）
```

### 3.3 关键操作与状态约束

| 操作 | 前置状态要求 |
|---|---|
| `ASSIGN`（分配负责人） | `PENDING_REVIEW` 或 `REJECTED` |
| `AUDIT_PASS`（审核通过） | `PENDING_REVIEW` |
| `AUDIT_REJECT`（审核拒绝） | `PENDING_REVIEW` |
| `PROMOTION_LINK`（转链） | `ASSIGNED` 或已是 `LINKED`（重新转链） |
| `ARCHIVE`（归档） | `LINKED` |
| `SELECT_TO_LIBRARY`（入选选品库） | `LINKED` |

## 4. 推广链接生成（转链）

### 4.1 入口条件

### 4.2 调用抖店 API 生成链接

```java

DouyinPromotionGateway.PromotionLinkResult result = douyinPromotionGateway.generateLink(

                finalExternalId,           // 外部唯一标识

```

### 4.3 pick_extra 构建

`buildPickExtra(userId)` 生成透传给抖店的附加参数，用于标识是哪个团长小助手账号发起的推广（归因用）：

```java

```

## 6. 活动与商品的关系

### 6.1 活动（ColonelsettlementActivity）

一个活动（推广计划）关联多个商品，商品通过 `activity_id` 找到对应的佣金率/服务费：

```java
// ColonelsettlementActivity.java
public class ColonelsettlementActivity {
    String activityId;           // 抖店活动 ID
    String name;               // 活动名称
    String activityType;       // 活动类型
    Long shopId;              // 商家店铺
    Long colonelBuyinId;      // 团长 Buyin ID（用于抖店原生归因）
    BigDecimal commissionRate; // 佣金率
    BigDecimal serviceRate;   // 服务费率
    LocalDateTime startTime;
    LocalDateTime endTime;
    Integer monthsOfProtection; // 保护期（月）
    Integer status;            // 1=进行中, 0=已结束
}
```

### 6.2 pick_extra 与 colonel_buyin_id 关联

推广链接的 `pick_extra` 参数中包含团长小助手账号标识，而 `colonel_buyin_id` 是抖店分配给团长的唯一 ID。两者配合完成归因。

## 7. 商品查询优化

### 7.1 两表分离查询

`ProductService` 的 `getPage()` 不直接 JOIN，而是分两步：

```java
// 步骤 1：查 ProductSnapshot（商品基础信息）
IPage<ProductSnapshot> snapshotPage = snapshotMapper.selectPage(...);

// 步骤 2：批量加载 ProductOperationState
Map<String, ProductOperationState> stateMap =
        loadOperationStatesForSnapshots(snapshots);  // 按 activityId+productId 做 key

// 步骤 3：批量加载 assignee 用户名
Map<UUID, String> assigneeNameMap = loadUserDisplayNames(Set.of(assigneeIds));

// 内存中合并
snapshot -> toLegacyProduct(snapshot, stateMap.get(key), assigneeNameMap)
```

这样做避免了 JOIN 开销，通过两次批量查询 + 内存拼接实现。

### 7.2 assign 场景下按负责人分页

```java
// getAssignedPickPage()：负责人维度分页
// 查 ProductOperationState 而非 ProductSnapshot
IPage<ProductOperationState> statePage = operationStateMapper.selectPage(
        new LambdaQueryWrapper<ProductOperationState>()

);
```
