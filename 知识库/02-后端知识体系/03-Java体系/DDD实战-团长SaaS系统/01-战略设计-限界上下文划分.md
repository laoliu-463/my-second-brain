# 01-战略设计-限界上下文划分

## 概述

团长 SaaS 系统按业务职责划分为 5 个限界上下文：Activity（活动领域）、Promotion Channel（推广通道路由）、Order（订单归因）、Settlement（结算分润）、Integration（集成层）。

## 各限界上下文职责

| 限界上下文 | 领域类型 | 职责 | 核心实体 |
|---|---|---|---|
| Activity Context | 核心域 CORE | 活动生命周期、商品入库、运营状态 | Activity、Product、OperationState |
| Promotion Channel Context | 核心域 CORE | 推广链接生成、pick_source/native 映射管理 | PickSourceMapping、PromotionLink |
| Order Context | 核心域 CORE | 订单同步、归因决策 | Order |
| Settlement Context | 支撑域 SUPPORTING | 结算分润计算 | Settlement、SettlementCycle |
| Integration Context | 通用域 GENERIC | 抖店 API 适配、协议转换 | DouyinApiClient、各类 Gateway |

## 上下文映射类型

| 关系 | 类型 | 说明 |
|---|---|---|
| Activity → Promotion Channel | **开放主机服务 + 发布订阅** | Activity 上下文通过事件通知 Promotion Channel |
| Promotion Channel → Integration | **防腐层（ACL）** | 通过 PromotionApi 适配器访问抖音，隔离外部依赖 |
| Order → Promotion Channel | **共享内核（Shared Kernel）** | 两上下文共享 PickSourceMapping 数据库表 |
| Settlement → Order | **遵循者（Conformist）** | Settlement 订阅 Order 的 AttributionResolved 事件 |
| Activity → Integration | **防腐层（ACL）** | ActivityApi 适配器隔离抖店 API 变化 |
