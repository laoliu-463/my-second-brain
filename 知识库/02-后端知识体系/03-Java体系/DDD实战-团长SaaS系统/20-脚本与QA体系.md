# 20-脚本与 QA 体系

项目脚本位于 `SAAS/scripts/`，全部为 PowerShell（`.ps1`）和 Node.js（`.cjs`）脚本，配合 Docker Compose 实现环境启停和 QA 回归。

## 1. 环境启停脚本

| 脚本 | 作用 |
|------|------|
| `start-test.ps1` | **主入口**：启动 TEST 环境（`docker-compose.yml + .env.test`），自动先停所有旧容器 |
| `start-real-pre.ps1` | **主入口**：启动 real-pre 环境（真实 SDK 联调），自动先停所有旧容器 |
| `start-test-all.ps1` | 兼容占位脚本，委托 `start-test.ps1` |
| `start-real-pre-docker.ps1` | 兼容占位脚本，委托 `start-real-pre.ps1` |
| `stop-all.ps1` | 停止所有 SAAS 容器（`saas-active` project），兼容清理旧 project 残留 |
| `stop-test-all.ps1` | 停止 test 环境容器，支持 `-RemoveVolumes` 删除数据卷 |
| `status.ps1` | 查看当前 SAAS 容器运行状态（容器名 + 启动时间 + 状态） |

```powershell
# 启动 TEST 环境
powershell.exe -File scripts/start-test.ps1

# 启动 real-pre 环境
powershell.exe -File scripts/start-real-pre.ps1

# 停止所有
powershell.exe -File scripts/stop-all.ps1

# 查看状态
powershell.exe -File scripts/status.ps1
```

核心命令：

```bash
# test 环境
docker compose --env-file .env.test --project-name saas-active -f docker-compose.yml up -d --build

# real-pre 环境
docker compose --env-file .env.real-pre --project-name saas-active -f docker-compose.yml up -d --build
```

## 2. 环境工具

| 脚本 | 作用 |
|------|------|
| `stack-utils.ps1` | **公共工具库**：被所有启停脚本引用，提供 `Read-EnvFile`、`Invoke-ComposeUp/Down` 等函数 |
| `mvn-test-docker.ps1` | 在 Docker 容器内运行 Maven 测试（使用同 backend 镜像/环境/卷） |
| `apply-test-db-patches.ps1` | 向 test PostgreSQL 容器执行 SQL 补丁 |
| `check-encoding.ps1` | 检查项目文件编码（UTF-8/GBK 等），排除噪音文件 |
| `check-env-topology.ps1` | 检查当前环境拓扑：端口占用、容器状态、profile 激活情况 |

## 3. QA 冒烟脚本

| 脚本 | 作用 |
|------|------|
| `qa-env.ps1` | 环境基础冒烟：健康检查、Mock 开关、匿名探针、登录态、Docker 容器状态 |
| `qa-api-smoke.ps1` | API 冒烟：调用 5 个核心接口（登录/达人/寄样/订单/Dashboard）验证响应 |
| `qa-page-smoke.ps1` | 页面冒烟：浏览器登录核心页面（Dashboard/达人/寄样/商品/订单），截图归档 |
| `qa-page-role.ps1` | 角色权限冒烟：验证不同角色登录后能看到的菜单和能访问的页面 |
| `qa-page-roles-all.ps1` | 遍历所有角色执行 `qa-page-role.ps1` |
| `qa-business-crud-flow.ps1` | 业务 CRUD 流程：登录→达人→寄样→商品→订单全链路操作验证 |
| `qa-business-browser-flow.ps1` | 浏览器业务流程：可见浏览器录制核心业务流程 |

## 4. Mock 数据与专项验证

| 脚本 | 作用 |
|------|------|
| `qa-mock-data-audit.ps1` | **Mock 数据审计**：核验 test 环境下各模块 mock 数据覆盖率和接口响应正确性 |
| `qa-test-mock-final.ps1` | TEST/mock 最终回归入口：整合 Mock 审计 + 编译 + 测试 + 语法检查，一键闭环 |

## 5. QA 子目录（`scripts/qa/`）

| 脚本 | 作用 |
|------|------|
| `run-real-pre-e2e.ps1` | real-pre 浏览器 E2E 回归 |
| `run-data-gap4-visible.ps1` | V2.2 Gap 4 数据看板增强可见浏览器回归 |
| `run-local-mock-supplement.ps1` | local-mock 交互级补充验收 |
| `run-qa-all.ps1` | 一键跑所有 QA 脚本（E2E + Gap4 + local-mock + 编译） |
| `watch-real-pre-order-settlements.ps1` | real-pre 多结算订单监听探针，持续轮询 `buyin.colonelMultiSettlementOrders` |
| `check-real-pre-real-data.ps1` | real-pre 真实数据复核：检查 active 表中是否有 mock/test/演示残留 |
| `run-real-pre-final-regression.ps1` | real-pre 主链路最终回归（健康/授权/活动/商品/订单/归因/Dashboard/Webhook） |
| `user-role-regression.js` | 用户角色回归（JS 版） |

## 6. 脚本分类速查

```
启停: start-test / start-real-pre / stop-all / status
工具: stack-utils(公共库) / check-env-topology / mvn-test-docker / apply-test-db-patches / check-encoding
QA:   qa-*(env/api/page/role/crud) / qa-mock-data-audit
回归: run-*-e2e / run-data-gap4 / run-qa-all
专项: watch-real-pre-order-settlements / check-real-pre-real-data
```

## 7. 一键命令汇总

```powershell
# TEST 环境冒烟（API + 页面 + 环境）
powershell.exe -File scripts/qa-env.ps1
powershell.exe -File scripts/qa-api-smoke.ps1
powershell.exe -File scripts/qa-page-smoke.ps1

# TEST/mock 最终回归
powershell.exe -File scripts/qa-test-mock-final.ps1

# real-pre E2E
powershell.exe -File scripts/qa/run-real-pre-e2e.ps1

# 一键跑所有
powershell.exe -File scripts/qa/run-qa-all.ps1

# 查看容器状态
powershell.exe -File scripts/status.ps1
```
