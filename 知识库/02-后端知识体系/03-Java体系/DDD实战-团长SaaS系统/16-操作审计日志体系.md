# 16-操作审计日志体系

## 1. 概述

团长 SaaS 的操作日志（`operation_log`）采用 **PostgreSQL 分区表**存储，按月自动创建分区，实现高效写入和低成本清理。所有用户操作（手动）和系统操作（自动）均通过 `OperationLogService.record()` 统一记录。

## 2. 分区表设计

### 2.1 分区策略

```
operation_log（父表）
       ├── op_log_2026_01（子表，月分区）
       ├── op_log_2026_02
       ├── op_log_2026_03
       └── ...
```

分区命名规则：`op_log_` + `yyyy_MM` 格式。

### 2.2 自动创建分区（auto-create partition）

`ensureLogPartition()` 在每次写入前检查目标分区是否存在，不存在则自动创建：

```java
// OperationLogService.java:175-202
private void ensureLogPartition(LocalDateTime createTime) {
    YearMonth month = YearMonth.from(target);
    String partitionName = "op_log_" + month.format(PARTITION_MONTH);

    if (!ENSURED_PARTITIONS.add(partitionName)) {
        return;  // 已创建过，直接返回
    }

    LocalDate start = month.atDay(1);
    LocalDate end = month.plusMonths(1).atDay(1);

    // ① 创建分区表
    String sql = String.format(
            "CREATE TABLE IF NOT EXISTS %s PARTITION OF operation_log FOR VALUES FROM ('%s') TO ('%s')",
            partitionName, start, end
    );
    jdbcTemplate.execute(sql);

    // ② 在该分区上建索引（每个分区独立索引）
    "CREATE INDEX IF NOT EXISTS idx_%s_create_time ON %s (create_time)";
    "CREATE INDEX IF NOT EXISTS idx_%s_user_id ON %s (user_id)";
    "CREATE INDEX IF NOT EXISTS idx_%s_module ON %s (module)";
}
```

### 2.3 分区裁剪（Partition Pruning）

由于按月分区，查询操作日志时带上时间范围（`create_time >= start AND create_time < end`），PostgreSQL 自动只扫描对应分区，避免全表扫描。

## 3. 幂等写入（DuplicateKey 防护）

### 3.1 eventKey 机制

`eventKey` 是每条日志的唯一标识，由事件类型 + 业务键拼接：

```java
// DouyinWebhookEventService.java:49
String eventKey = buildEventKey(eventType, parsed.payload(), body);
```

### 3.2 幂等插入

`captureColonelOpenEvent()` 在插入前先查 `eventKey` 是否存在：

```java
// DouyinWebhookEventService.java:51-54
DouyinWebhookEvent existing = findByEventKey(eventKey);
if (existing != null) {
    return new CaptureResult(existing.getId(), existing.getEventType(), existing.getStatus(), true);
}
// 不存在 → 插入新记录
```

通过唯一索引（或主键）保证幂等，重复推送的 Webhook 不会产生重复记录。

## 4. 数据保留与清理

### 4.1 过期分区删除

`cleanupOldPartitions(retentionDays)` 按月粒度删除过期分区：

```java
// OperationLogService.java:211-229
@Transactional
public int cleanupOldPartitions(int retentionDays) {
    YearMonth cutoff = YearMonth.from(LocalDate.now().minusDays(days));
    List<String> partitions = listOperationLogPartitions();  // 查 pg_inherits
    for (String name : partitions) {
        YearMonth month = parsePartitionMonth(name);  // 从 op_log_2026_01 解析
        if (month.isBefore(cutoff)) {
            jdbcTemplate.execute("DROP TABLE IF EXISTS " + name);  // 删除分区
            dropped++;
        }
    }
}
```

### 4.2 清理策略对比

| 方式 | 优点 | 缺点 |
|---|---|---|
| `DROP PARTITION`（当前方案） | 极快，O(1)，不触发 VACUUM | 按月整删，不可逆 |
| `DELETE FROM partition WHERE create_time < cutoff` | 灵活，可按日清理 | 慢，产生大量 dead tuples |

当前方案适合审计日志场景（按月保留，定期整月删除）。

## 5. 日志内容字段

### 5.1 record() 写入字段

```java
INSERT INTO operation_log (
    id, user_id, username,
    module, action,                // 模块 + 操作类型
    target_type, target_id, target_name, content,  // 操作对象
    request_method, request_url, request_params, request_body,  // HTTP 请求信息
    response_code, response_body,   // HTTP 响应
    ip_address, user_agent, duration_ms, error_message,
    create_time, deleted
)
```

### 5.2 recordSystemAction() 系统操作

用于非 HTTP 触发的系统内部操作（如定时任务、消息队列）：

```java
recordSystemAction(
    operatorId,    // 操作人 ID（系统操作可传 null）
    module,        // "达人管理"
    action,        // "归属覆盖"
    requestMethod, // "SYSTEM"
    targetType,    // "Talent"
    targetId,      // 达人 UUID
    targetName,    // 达人昵称
    content        // "强制分配给用户 XXX"
);
```

## 6. 日志查询（findPage）

```java
// OperationLogService.java:124-154
public IPage<OperationLog> findPage(...) {
    // 默认查最近 89 天数据
    LocalDateTime start = (startDate == null ? LocalDate.now().minusDays(89) : startDate).atStartOfDay();
    LocalDateTime end = (endDate == null ? LocalDate.now().plusDays(1) : endDate.plusDays(1)).atStartOfDay();
    // 支持按 module/action/username/requestMethod 过滤
    // 按 create_time 倒序
}
```

## 7. JSONB 字段存储

`request_params`、`request_body`、`response_body` 使用 PostgreSQL JSONB 类型存储：

```java
// CAST(? AS jsonb) 将 Java 对象序列化为 JSONB 存入
jdbcTemplate.update(sql, ..., toJson(log.getRequestParams()), toJson(log.getRequestBody()), ...);
```

支持直接对 JSONB 字段内联查询（如 `extra_data->>'talent_uid'`）。
