# 07-样品生命周期

## 1. 概述

样品（Sample）是团长 SaaS 的核心业务实体之一。团长向商家申请样品用于体验和推广，样品从申请到完成有一套完整的状态机管理，包括审批、发货、交付、评价等阶段。

**核心业务逻辑**：
- 样品申请需审核（待审核 → 通过/拒绝）
- 通过后等待发货（待发货）
- 发货后等待达人确认收货（待收货/待交付）
- 达人需在规定时间内完成"作业"（出单）才能结单（待作业 → 已完成）
- 超时未发货或超时未完成作业，系统自动关闭

## 2. 样品状态机

### 2.1 状态定义

| 状态值 | 名称 | 说明 |
|---|---|---|
| 1 | `PENDING_AUDIT` | 待审核 |
| 2 | `PENDING_SHIP` | 待发货 |
| 3 | `PENDING_RECEIVE` | 待收货（已发货） |
| 4 | `PENDING_DELIVER` | 待交付（已发货待确认） |
| 5 | `PENDING_HOMEWORK` | 待作业（达人需在规定天数内出单） |
| 6 | `COMPLETED` | 已完成 |
| 7 | `REJECTED` | 已拒绝 |
| 8 | `CLOSED` | 已关闭 |

### 2.2 状态流转图

```
申请人提交申请
      ↓
  [1] 待审核 ───(拒绝)──→ [7] 已拒绝
      ↓(通过)
  [2] 待发货 ───(超时自动关闭)──→ [8] 已关闭
      ↓(发货)
  [3] 待收货 ───(确认收货)──→ [4] 待交付
                               ↓(交付确认)
                          [5] 待作业 ───(超时自动关闭)──→ [8] 已关闭
                              ↓(出单匹配订单)
                         [6] 已完成
```

### 2.3 状态流转约束

关键流转约束（`SampleRequest.java` 字段）：

| 字段 | 说明 |
|---|---|
| `audit_time` | 审核时间（通过/拒绝时间） |
| `ship_time` | 发货时间 |
| `deliver_time` | 交付确认时间（达人确认收货） |
| `complete_time` | 完成时间（出单后系统自动更新） |
| `close_time` / `close_reason` | 关闭时间和原因 |
| `tracking_no` | 快递单号 |

## 3. 超时自动关闭机制

### 3.1 两类超时

| 超时类型 | 触发条件 | 关闭状态 | 超时时间来源 |
|---|---|---|---|
| 待发货超时 | 审核通过后超过 N 天未发货 | `PENDING_SHIP(2)` → `CLOSED(8)` | `businessRuleConfigService.getSampleTimeoutPendingShipDays()` |
| 待作业超时 | 交付确认后超过 N 天未出单 | `PENDING_HOMEWORK(5)` → `CLOSED(8)` | `businessRuleConfigService.getSampleTimeoutHomeworkDays()` |

### 3.2 定时 Job

`SampleLifecycleJob` 每天凌晨 2 点执行（`@Scheduled(cron = "0 0 2 * * ?")`）：

```java
// SampleLifecycleJob.java:36-58
@Scheduled(cron = "0 0 2 * * ?")
public void autoCloseTimeoutRequests() {
    if (!acquireJobLock()) {  // Redis 分布式锁
        log.info("SampleLifecycleJob skipped, another process is running");
        return;
    }
    try {
        int closedHomework = sampleLifecycleService.autoCloseTimeoutPendingHomework();
        int closedShip = sampleLifecycleService.autoCloseTimeoutPendingShip();
    } finally {
        releaseJobLock();
    }
}
```

**待发货超时**：

```sql
FROM sample_request sr
```

```java
// SampleLifecycleService.java:189-199
private String resolveTalentUid(ColonelsettlementOrder order) {
    Map<String, Object> extra = order.getExtraData();
    String talentUid = asText(extra.get("talent_uid"));
    if (StringUtils.hasText(talentUid)) return talentUid;
```

## 5. 达人资格校验

### 5.1 校验维度

`SampleEligibilityService.evaluate()` 在达人申请样品时校验两个维度：

| 维度 | 配置来源 | 说明 |
|---|---|---|
| 近30天销售额 | `sample_default_standard.min_30_day_sales` | 达人近30天销售额需达到门槛 |
| 达人等级 | `sample_default_standard.min_level` | LV0/LV1/LV2 |

### 5.2 等级判定规则

**优先级**：Talent 表 > 粉丝数推算

```java
// SampleEligibilityService.java:64-78
private String resolveLevel(Talent talent, CrawlerTalentInfo talentInfo) {
    // 1. 直接取 Talent.level
    if (StringUtils.hasText(talent.getLevel())) {
        return normalizeLevel(talent.getLevel());
    }
    // 2. 根据粉丝数推算
    long fans = talent.getFans() != null ? talent.getFans()
            : talentInfo.getFansCount() != null ? talentInfo.getFansCount() : 0;
    if (fans >= 300_000) return "LV2";
    if (fans >= 100_000) return "LV1";
    return "LV0";
}

// 等级标准化（normalizeLevel）
"A" / "S" → "LV2"
"B"       → "LV1"
其他      → "LV0"
```

## 6. 状态日志

每条样品的状态流转都会记录到 `sample_status_log` 表（`SampleStatusLogService`）：

