# 09-达人管理与资格体系

## 1. 概述

达人（Talent）是团长 SaaS 的核心业务实体之一。团长通过平台管理达人资源，对其进行认领、分配、保护和数据补全。

达人体系的核心矛盾是：**公海私海分离**（避免销售撞单）+ **数据补全**（达人信息从爬虫采集到手动补充）+ **保护期机制**（认领后一段时间内独占）。

## 3. 公海私海分离机制

### 3.1 分界规则

```
公海（PUBLIC）：
  - 无活跃认领记录（status != ACTIVE 或无记录）
  - 所有用户可见

私海（PRIVATE）：
  - 有活跃认领记录（status == ACTIVE）
  - 仅认领人及其同部门人可见
```

### 3.2 公海池（Public Pool）

```java
// TalentService.java:103-114
public List<Talent> getPublicPool() {
    Set<UUID> claimedTalentIds = getClaimedTalentIds();  // 所有有活跃认领的达人
    return talentMapper.selectList(
            .eq(status, 1)
            .ne(blacklisted, true))
            .stream()
            .filter(talent -> !claimedTalentIds.contains(talent.getId()))  // 排除私海
            .sorted(fans 降序)
            .limit(500)  // 公海最多返回 500 条
            .toList();
}
```

### 3.3 私海池（Private Pool）

```java
// TalentService.java:116-125
public List<Talent> getPrivatePool(UUID userId) {
    List<TalentClaim> claims = talentClaimMapper.findActiveByUserId(userId);
    Set<UUID> talentIds = claims.stream().map(TalentClaim::getTalentId).toSet();
    return talentMapper.selectBatchIds(talentIds).stream()
            .limit(500)
            .toList();
}
```

private void assertCanAccess(Talent talent, UUID currentUserId, UUID currentDeptId, DataScope dataScope) {
    List<TalentClaim> activeClaims = talentClaimMapper.findActiveByTalentId(talent.getId());
    if (dataScope == DataScope.PERSONAL) {
        boolean owned = activeClaims.stream().anyMatch(c -> currentUserId.equals(c.getUserId()));
        if (!owned) throw new ForbiddenException("无权查看该达人详情");
    } else if (dataScope == DataScope.DEPT) {
        boolean owned = activeClaims.stream().anyMatch(c -> currentDeptId.equals(c.getDeptId()));
        if (!owned) throw new ForbiddenException("无权查看该达人详情");
    }
}

## 4. 认领与保护期机制

### 4.1 认领流程（claim）

```java
// TalentService.java:289-341
@Transactional
public Talent claim(UUID talentId, UUID userId, UUID deptId) {
    // 1. Redis 分布式锁防并发抢单（10 秒自动过期）
    Boolean locked = redisTemplate.opsForValue().setIfAbsent(lockKey, userId, 10, TimeUnit.SECONDS);
    if (!locked) throw new BusinessException("达人认领处理中，请稍后重试");

    // 2. 保护期检查：自己已认领则报错
    TalentClaim selfActive = talentClaimMapper.findActiveByTalentAndUser(talentId, userId);
    if (selfActive != null) throw new BusinessException("你已认领该达人，无需重复认领");

    // 3. 写入/更新 TalentClaim（status=ACTIVE）
    claim.setProtectedUntil(now.plusDays(protectDays));  // 保护期 = 今天 + N 天

    // 4. 更新 Talent.ownerId
    talent.setOwnerId(userId);
    talent.setClaimedAt(now);
    talentMapper.updateById(talent);
}
```

### 4.2 保护期机制

保护期时长从 `BusinessRuleConfigService.getTalentProtectionDays()` 读取（配置项）。

保护期内：
- 其他用户无法认领该达人（`findActiveByTalentAndUser` 会查到活跃认领）
- 达人显示为"私海"

保护期到期后：
- 认领状态变为 `EXPIRED`（不自动删除记录）
- 达人退回公海，其他人可以认领

### 4.3 释放（release）

```java
// TalentService.java:343-371
@Transactional
public Talent release(UUID talentId, UUID userId, UUID deptId, Collection<?> roleCodes) {
    // 1. 查找该达人的所有活跃认领
    List<TalentClaim> activeClaims = talentClaimMapper.findActiveByTalentId(talentId);

    // 2. 权限检查：认领人本人 或 admin 角色
    boolean isAdmin = hasRole(roleCodes, "admin");
    TalentClaim releaseTarget = activeClaims.stream()
            .filter(claim -> canRelease(claim, userId, deptId, isAdmin))
            .findFirst()
            .orElseThrow(() -> new BusinessException("仅认领人或管理员可以释放达人"));

    // 3. 标记为已释放
    releaseTarget.setStatus(CLAIM_STATUS_RELEASED);
    releaseTarget.setProtectedUntil(LocalDateTime.now());  // 立即过期
    talentClaimMapper.updateById(releaseTarget);

    // 4. 清除 Talent.ownerId
    talent.setOwnerId(null);
    talentMapper.updateById(talent);
}
```

## 5. 达人数据补全（Enrich）

### 5.1 补全状态机

```
创建达人 → enrichStatus = PENDING
         → enrich() 执行
              ├── 成功 → SUCCESS（自动补充 nickname/fans/level 等字段）
              ├── 失败 → FAILED（异常）
              └── 部分成功 → WAIT_MANUAL（需人工补充）
```

### 5.2 补全流程（create 时触发）

```java
// TalentService.java:233-252
TalentEnrichTask task = createEnrichTask(request, ENRICH_TASK_STATUS_RUNNING, null);
try {
    TalentEnrichOrchestrator.OrchestrateResult result =
            talentEnrichOrchestrator.enrich(request, false);  // false = 非强制刷新
    enrichTalentInfo(request, false);  // 公开页爬虫（可选）
    talentMapper.updateById(request);

    if (result.updated()) {
        markEnrichTask(task, ENRICH_TASK_STATUS_SUCCESS, null);
    } else {
        request.setEnrichStatus(ENRICH_TASK_STATUS_WAIT_MANUAL);
        markEnrichTask(task, ENRICH_TASK_STATUS_WAIT_MANUAL, result.message());
    }
} catch (RuntimeException ex) {
    markEnrichTask(task, ENRICH_TASK_STATUS_FAILED, ex.getMessage());
    throw ex;
}
```

## 6. 达人详情页批量查询优化

`TalentQueryService.detail()` 查询达人详情时涉及多表关联，系统采用分批查询避免 N+1：

### 6.1 批量加载模式

```java
// TalentQueryService.java:220-262
private void enrichTalentCards(List<Talent> talents, UUID currentUserId) {
    // 一次性加载所有达人的认领记录
    List<TalentClaim> claims = talentClaimMapper.selectList(
            .in(TalentClaim::getTalentId, talentIds)
            .eq(TalentClaim::getDeleted, 0)
            .orderByDesc(TalentClaim::getClaimedAt));

    // 一次性加载所有认领人信息
    Map<UUID, SysUser> ownerMap = loadOwnerMap(claims);  // selectBatchIds

    // 按达人 ID 分组，内存中合并
    Map<UUID, List<TalentClaim>> byTalent = claims.stream()
            .collect(Collectors.groupingBy(TalentClaim::getTalentId));
}
```

### 6.2 IN 分批查询

`loadSampleCounts()` 和 `loadOrderAggregates()` 对大批量使用分批 IN 查询（每批 200 条）：

```java
// TalentQueryService.java:332-342
private <T> List<List<T>> partition(Collection<T> values, int batchSize) {
    List<T> list = new ArrayList<>(values);
    List<List<T>> partitions = new ArrayList<>();
    for (int index = 0; index < list.size(); index += batchSize) {
        partitions.add(list.subList(index, Math.min(index + batchSize, list.size())));
    }
    return partitions;
}
```

## 8. 达人等级与档位

### 8.1 等级推算（来自爬虫或粉丝数）

| 等级 | 条件 | 说明 |
|---|---|---|
| LV0 | 默认 | 粉丝 < 10 万 |
| LV1 | 粉丝 ≥ 10 万 | B 级达人 |
| LV2 | 粉丝 ≥ 30 万 | A/S 级达人 |

等级字段标准化（`normalizeLevel`）："A"/"S" → LV2，"B" → LV1，其他 → LV0

### 8.2 档位计算

达人详情页展示多个档位指标（用于筛选和展示）：

| 档位字段 | 计算方式 |
|---|---|
| `liveSalesBand` | 近30天销售额分档 |
| `liveViewBand` | 粉丝数分档 |
| `liveGpmBand` | 场均 GPM（GPM = 销售额/观看人数） |
| `videoSalesBand` | 平均单视频销售额 |
| `videoPlayBand` | 预估播放量档位 |
