**COALESCE UPSERT 问题（Bug 根因）**：

```java
// ColonelsettlementActivityMapper.xml 或 @Select 注解中的 UPSERT
@Select("""
    INSERT INTO colonel_activity
        (id, activity_id, colonel_buyin_id, extra_data, ...)
    VALUES
        (#{id}, #{activityId}, #{colonelBuyinId}, #{extraData}::jsonb, ...)
    ON CONFLICT (activity_id)
    DO UPDATE SET
        colonel_buyin_id = COALESCE(
            colonel_activity.colonel_buyin_id,   -- 旧值优先，永不覆盖
            EXCLUDED.colonel_buyin_id            -- 只在新值为非null时写入
        ),
        extra_data = COALESCE(
            colonel_activity.extra_data,
            EXCLUDED.extra_data
        )
    """)
void upsertRealActivityMeta(...);
```

时序场景：

| 场景 | 旧主字段 | 新值 | 写入结果 |
|---|---|---|---|
| 首次 UPSERT（API 有值） | null | 719302412345678901 | **719302412345678901** ✅ |
| 更新其他字段（buyinId传null） | 719302412345678901 | null | **719302412345678901** ✅（旧值保留）|
| 首次 UPSERT 时 API 值未传入 | **null** | null | **null 永久** ❌ |

`extra_data` JSONB 在所有场景下均正确写入（含真实 colonel_buyin_id），但旧代码只查主字段，导致第3层解析返回 null。

### 领域服务：ColonelBuyinIdResolver（分层解析策略 POLICY）

实现在 `ProductService.java` 内部，作为 private 方法私有于 Service 层。源码使用 Java record `NativeColonelBuyinResolution` 封装解析结果。

```java
// ProductService.java:1364-1380（真实 record 声明）
private record NativeColonelBuyinResolution(
        String colonelBuyinId,
        NativeColonelBuyinSource source   // 标注来源层，用于日志
) {
    private static NativeColonelBuyinResolution unresolved() {
        return new NativeColonelBuyinResolution(null, NativeColonelBuyinSource.UNRESOLVED);
    }
    private boolean resolved() {
        return StringUtils.hasText(colonelBuyinId);
    }
}

private enum NativeColonelBuyinSource {
    PRODUCT_SNAPSHOT,         // 第1层：ProductSnapshot.rawPayload.extraData
    PRODUCT_OPERATION_STATE,  // 第2层：ProductOperationState.auditPayload
    COLONEL_ACTIVITY,         // 第3层：colonel_activity 主字段 + extra_data（patch 修复点）
    ACTIVITY_API_DETAIL,      // 第4层：activityApi.detail() 回填
    UNRESOLVED               // 全部未命中
}
```

**完整4层解析入口（源码）：**

```java
// ProductService.java:1101-1119（真实源码）
/**
 * 分层解析 colonelBuyinId，从本地到远程依次尝试：
 *   ① ProductSnapshot.rawPayload → extraData.colonelBuyinId
 *   ② ProductOperationState.auditPayload → extraData.colonelBuyinId
 *   ③ colonelsettlementActivity 主字段 + extraData（patch 新增 fallback）
 *   ④ activityApi.detail() 回填（异常被吞，不阻断转链主流程）
 */
private NativeColonelBuyinResolution
resolveColonelBuyinIdForNativeMapping(String activityId, String productId) {

    // 第1层：ProductSnapshot.rawPayload
    String colonelBuyinId = resolveColonelBuyinIdFromSnapshot(activityId, productId);
    if (StringUtils.hasText(colonelBuyinId))
        return new NativeColonelBuyinResolution(colonelBuyinId, NativeColonelBuyinSource.PRODUCT_SNAPSHOT);

    // 第2层：ProductOperationState.auditPayload
    colonelBuyinId = resolveColonelBuyinIdFromOperationState(activityId, productId);
    if (StringUtils.hasText(colonelBuyinId))
        return new NativeColonelBuyinResolution(colonelBuyinId, NativeColonelBuyinSource.PRODUCT_OPERATION_STATE);

    // 第3层：colonel_activity 主字段 + extraData
    colonelBuyinId = resolveColonelBuyinIdFromActivity(activityId);
    if (StringUtils.hasText(colonelBuyinId))
        return new NativeColonelBuyinResolution(colonelBuyinId, NativeColonelBuyinSource.COLONEL_ACTIVITY);

    // 第4层：API 回填
    colonelBuyinId = hydrateColonelActivityMeta(activityId);
    if (StringUtils.hasText(colonelBuyinId))
        return new NativeColonelBuyinResolution(colonelBuyinId, NativeColonelBuyinSource.ACTIVITY_API_DETAIL);

    return NativeColonelBuyinResolution.unresolved();
}
```

**第3层源码（BUG 修复点）：**

```java
// ProductService.java:1121-1141（真实源码）
/**
 * 从 colonel_activity 表解析 colonelBuyinId。
 * ① 优先查主字段 colonel_buyin_id
 * ② 主字段为 null 时，fallback 查 extra_data JSONB（patch 新增）
 */
private String resolveColonelBuyinIdFromActivity(String activityId) {
    if (!StringUtils.hasText(activityId)) return null;

    // ① 主字段查询（可能被 COALESCE UPSERT 置为 null）
    ColonelsettlementActivity activity =
            colonelActivityMapper.selectByActivityId(activityId);
    if (activity != null && activity.getColonelBuyinId() != null)
        return String.valueOf(activity.getColonelBuyinId());

    // ② extraData JSONB fallback（patch 新增）
    // hydrateColonelActivityMeta() 已将真实 colonel_buyin_id 写入 extraData
    Map<String, Object> extraData =
            colonelActivityMapper.selectExtraDataByActivityId(activityId);
    if (extraData != null && !extraData.isEmpty()) {
        String fromExtra = firstNonBlank(
                readString(extraData, "colonel_buyin_id", "colonelBuyinId"),
                readString(
                        readNestedMap(extraData, "extra_data", "extraData"),
                        "colonel_buyin_id", "colonelBuyinId"));
        if (StringUtils.hasText(fromExtra)) return fromExtra;
    }
    return null;
}
```

**第1层源码（JSON + 正则双重兜底）：**

```java
// ProductService.java:1177-1194 + 1241-1271（真实源码）
// ① JSON 结构解析（支持 extra_data 嵌套）
private String resolveColonelBuyinIdFromSnapshot(String activityId, String productId) {
    ProductSnapshot snapshot = snapshotMapper.selectOne(
            new LambdaQueryWrapper<ProductSnapshot>()
                    .eq(ProductSnapshot::getActivityId, activityId)
                    .eq(ProductSnapshot::getProductId, productId)
                    .eq(ProductSnapshot::getDeleted, 0)
                    .last("limit 1"));
    if (snapshot == null) return null;

    String fromPayload = resolveColonelBuyinIdFromPayload(snapshot.getRawPayload());
    if (StringUtils.hasText(fromPayload)) return fromPayload;

    // ② 正则兜底（非 JSON 原始文本）
    return extractColonelBuyinIdFromText(snapshot.getRawPayload());
}

// 从 JSON payload 提取（支持 extra_data 嵌套 + 多字段名变体）
private String resolveColonelBuyinIdFromPayload(String raw) {
    Map<String, Object> payload = parseJsonObject(raw);
    if (payload.isEmpty()) return null;

    String fromRoot = readString(payload,
            "origin_colonel_buyin_id", "originColonelBuyinId",
            "colonel_buyin_id", "colonelBuyinId");
    if (StringUtils.hasText(fromRoot)) return fromRoot;

    Map<String, Object> extraData =
            readNestedMap(payload, "extra_data", "extraData");
    return readString(extraData,
            "origin_colonel_buyin_id", "originColonelBuyinId",
            "colonel_buyin_id", "colonelBuyinId");
}

// 正则兜底：从原始文本（非 JSON）中提取 10+ 位数字
private static final Pattern BUYIN_ID_PATTERN = Pattern.compile(
    "(?:origin_colonel_buyin_id|originColonelBuyinId|" +
    "colonel_buyin_id|colonelBuyinId)\\s*[=:]\\s*['\"]?([0-9]{10,})"
);
private String extractColonelBuyinIdFromText(String raw) {
    if (!StringUtils.hasText(raw)) return null;
    Matcher matcher = BUYIN_ID_PATTERN.matcher(raw);
    if (matcher.find()) return matcher.group(1);
    return null;
}
```

**第4层兜底源码（API 回填）：**

```java
// ProductService.java:1143-1175（真实源码）
/**
 * 调抖音 enterprise.marketing.promotion.activity.detail API，
 * 将返回的 colonel_buyin_id 回填到 colonel_activity 表（UPSERT）。
 * 失败不阻断转链主流程（异常被吞）。
 */
private String hydrateColonelActivityMeta(String activityId) {
    if (!StringUtils.hasText(activityId)) return null;
    try {
        Map<String, Object> response = activityApi.detail(null, activityId);
        Map<String, Object> data = readPrimaryDataNode(response);
        Long colonelBuyinId = readLong(data,
                "colonel_buyin_id", "colonelBuyinId");
        if (colonelBuyinId == null || colonelBuyinId <= 0L) return null;

        colonelActivityMapper.upsertRealActivityMeta(
                UUID.nameUUIDFromBytes(
                        ("real-activity-" + activityId).getBytes(StandardCharsets.UTF_8)),
                activityId,
                readString(data, "activity_name", "activityName", "name"),
                readLong(data, "shop_id", "shopId"),
                readString(data, "shop_name", "shopName"),
                colonelBuyinId,
                readDecimal(data, "commission_rate", "commissionRate"),
                readDecimal(data, "service_rate", "serviceRate"),
                parseDateTimeValue(readString(data,
                        "activity_start_time", "activityStartTime",
                        "start_time", "startTime")),
                parseDateTimeValue(readString(data,
                        "activity_end_time", "activityEndTime",
                        "end_time", "endTime")),
                readString(data, "status_text", "statusText", "status"),
                LocalDateTime.now(),
                data   // ← 完整 JSON 写入 extra_data（含真实 colonel_buyin_id）
        );
        return String.valueOf(colonelBuyinId);
    } catch (Exception ex) {
        log.warn("Hydrate colonel activity meta failed, activityId={}", activityId, ex);
        return null;  // 不阻断转链
    }
}
```

```java
// ProductService.java:1073-1084（真实源码）
/**
 * 构建 pickExtra：抖店用来区分达人的标识字段（最长 20 字符）。
 * 优先用用户的 channelCode，兜底用 userId。
 */
private String buildPickExtra(UUID userId) {
    if (userId == null) return null;
    SysUser user = sysUserMapper.selectById(userId);
    if (user != null && StringUtils.hasText(user.getChannelCode())) {
        String candidate = "channel_" + user.getChannelCode().trim().toLowerCase(Locale.ROOT);
        return candidate.length() <= 20 ? candidate
                : candidate.substring(0, 20);
    }
    String fallback = "channel_" + userId.toString().replace("-", "");
    return fallback.substring(0, Math.min(fallback.length(), 20));
}
```

### 领域服务：PickSourceMappingService（溯源映射仓储，真实 upsert 实现）

```java
// PickSourceMappingService.java:1-452（真实源码，核心方法）
@Service
public class PickSourceMappingService {

    public static final String SOURCE_TYPE_NATIVE = "NATIVE";
    public static final String SOURCE_TYPE_PICK_SOURCE = "PICK_SOURCE";
    private static final int validMonths = 3;

    // 5个重载方法 → 全参方法（第6个）
    @Transactional(rollbackFor = Exception.class)
    public void saveOrUpdate(UUID userId, String channelUserName, UUID deptId,
            String talentId, String talentName, String shortId, UUID uuidSeed,
            String pickSource, String productId, String activityId, String sourceUrl,
            String convertedUrl, UUID promotionLinkId, String scene, String pickExtra,
            String colonelBuyinId, String sourceType) {

        // ① 解析 sourceType（优先级：显式参数 > colonelBuyinId 有无）
        String resolvedSourceType = resolveSourceType(sourceType, colonelBuyinId);

        // ② 查询已存在 mapping（4级查询优先级）
        PickSourceMapping existing = findExistingMapping(
                userId, pickSource, productId, activityId, promotionLinkId,
                colonelBuyinId, resolvedSourceType);

        if (existing == null) {
            // ③ INSERT
            try {
                PickSourceMapping mapping = buildMapping(...);
                mapping.setColonelBuyinId(resolveColonelBuyinId(colonelBuyinId));
                pickSourceMappingMapper.insert(mapping);
                logNativeAmbiguousIfNeeded(mapping);
                return;
            } catch (DuplicateKeyException ex) {
                // ④ 并发保护：两个线程同时 INSERT → 重查
                existing = findExistingMapping(...);
                if (existing == null) throw ex;
            }
        }

        // ⑤ UPDATE 已存在记录
        existing.setUserId(userId);
        existing.setColonelBuyinId(resolveColonelBuyinId(colonelBuyinId));
        existing.setSourceType(resolvedSourceType);
        existing.setValidUntil(LocalDateTime.now().plusMonths(validMonths));
        existing.setStatus(1);
        pickSourceMappingMapper.updateById(existing);
        logNativeAmbiguousIfNeeded(existing);
    }
}
```

**findExistingMapping 查询优先级（真实源码）：**

```java
// PickSourceMappingService.java:279-328（真实源码）
private PickSourceMapping findExistingMapping(UUID userId, String pickSource,
        String productId, String activityId, UUID promotionLinkId,
        String colonelBuyinId, String sourceType) {

    // 优先级1：promotionLinkId
    if (promotionLinkId != null) {
        PickSourceMapping m = pickSourceMappingMapper.selectOne(
                new LambdaQueryWrapper<PickSourceMapping>()
                        .eq(PickSourceMapping::getPromotionLinkId, promotionLinkId)
                        .last("limit 1"));
        if (m != null) return m;
    }

    // 优先级2：(userId + pickSource + productId + activityId)
    if (userId != null && StringUtils.hasText(pickSource)
            && StringUtils.hasText(productId) && StringUtils.hasText(activityId)) {
        PickSourceMapping m = pickSourceMappingMapper.selectOne(...); // 条件省略
        if (m != null) return m;
    }

    // 优先级3：(NATIVE + colonelBuyinId + productId + activityId + userId)
    //   这是 NATIVE 通道的核心唯一性约束
    if (SOURCE_TYPE_NATIVE.equals(sourceType)
            && userId != null
            && StringUtils.hasText(colonelBuyinId)
            && StringUtils.hasText(productId)
            && StringUtils.hasText(activityId)) {
        PickSourceMapping m = pickSourceMappingMapper.selectOne(
                new LambdaQueryWrapper<PickSourceMapping>()
                        .eq(PickSourceMapping::getColonelBuyinId, colonelBuyinId)
                        .eq(PickSourceMapping::getProductId, productId)
                        .eq(PickSourceMapping::getActivityId, activityId)
                        .eq(PickSourceMapping::getUserId, userId)
                        .eq(PickSourceMapping::getSourceType, SOURCE_TYPE_NATIVE)
                        .last("limit 1"));
        if (m != null) return m;
    }

    // 优先级4：纯 pickSource 兜底
    if (StringUtils.hasText(pickSource)
            && !StringUtils.hasText(productId)
            && !StringUtils.hasText(activityId)) {
        return pickSourceMappingMapper.selectOne(
                new LambdaQueryWrapper<PickSourceMapping>()
                        .eq(PickSourceMapping::getPickSource, pickSource)
                        .last("limit 1"));
    }
    return null;
}
```

**logNativeAmbiguousIfNeeded（AMBIGUOUS 检测，真实源码）：**

```java
// PickSourceMappingService.java:337-366（真实源码）
/**
 * 写完 NATIVE mapping 后，检查同一 (colonelBuyinId, activityId, productId)
 * 是否存在多个不同的 userId → 标记为歧义（AMBIGUOUS）。
 * 两人操作同一商品时，两人的订单都不归因。
 */
private void logNativeAmbiguousIfNeeded(PickSourceMapping mapping) {
    if (mapping == null
            || !SOURCE_TYPE_NATIVE.equals(mapping.getSourceType())
            || !StringUtils.hasText(mapping.getColonelBuyinId())
            || !StringUtils.hasText(mapping.getActivityId())
            || !StringUtils.hasText(mapping.getProductId()))
        return;

    List<PickSourceMapping> mappings = pickSourceMappingMapper.selectList(
            new LambdaQueryWrapper<PickSourceMapping>()
                    .eq(PickSourceMapping::getSourceType, SOURCE_TYPE_NATIVE)
                    .eq(PickSourceMapping::getColonelBuyinId, mapping.getColonelBuyinId())
                    .eq(PickSourceMapping::getActivityId, mapping.getActivityId())
                    .eq(PickSourceMapping::getProductId, mapping.getProductId())
                    .eq(PickSourceMapping::getStatus, 1));

    if (mappings == null || mappings.isEmpty()) return;

    long distinctUsers = mappings.stream()
            .map(PickSourceMapping::getUserId)
            .filter(java.util.Objects::nonNull)
            .distinct()
            .count();

    if (distinctUsers > 1) {
        log.warn("Native mapping ambiguous for activityId={}, productId={}, " +
                "colonelBuyinId={}, distinctUsers={}",
                mapping.getActivityId(), mapping.getProductId(),
                mapping.getColonelBuyinId(), distinctUsers);
    }
}
```

### 领域服务：AttributionPolicy（归因策略 POLICY）

```java
// 领域服务：AttributionPolicy
// POLICY（策略模式）：订单归因优先级链
public class AttributionPolicy {

    public AttributionResult resolve(Order order) {
        // 第1优先：独家商家归因
        ExclusiveOwner merchant = findExclusiveMerchantOwner(order.getShopId());
        if (merchant != null)
            return attributed(merchant.userId(), REASON_ATTRIBUTED);

        // 第2优先：独家达人归因
        ExclusiveOwner talent = findExclusiveTalentOwner(order.getTalentUid());
        if (talent != null)
            return attributed(talent.userId(), REASON_ATTRIBUTED);

        // 第3优先：抖店原生归因（colonel_order_info 通道）
        if (hasColonelOrderInfo(order)) {
            NativeResolution result = resolveNativeColonel(order);
            if (result.mapping() != null)
                return attributed(result.mapping().getUserId(), REASON_COLONEL_ORDER_INFO);
            return unattributed(result.reason());
        }

        // 第4优先：渠道归因（pick_source 通道）
        if (hasPickSource(order)) {
            PickSourceMapping mapping = findPickSourceMapping(order);
            if (mapping != null)
                return attributed(mapping.getUserId(), REASON_ATTRIBUTED);
            return unattributed(REASON_MAPPING_NOT_FOUND);
        }

        // 兜底：无归因字段
        return unattributed(REASON_NO_PICK_SOURCE);
    }

    // 抖店原生归因精确查询
    private NativeResolution resolveNativeColonel(Order order) {
        String buyinId = order.getColonelOrderInfo().getColonelBuyinId();
        String activityId = order.getColonelOrderInfo().getActivityId();
        String productId = order.getProductId();

        // 精确查询 native mapping
        // WHERE colonel_buyin_id = buyinId
        // AND activity_id = activityId
        // AND product_id = productId
        // AND source_type = 'NATIVE'
        // AND status = 1
        List<PickSourceMapping> mappings = pickSourceMappingMapper.selectList(
            eq(PickSourceMapping::getColonelBuyinId, buyinId),
            eq(PickSourceMapping::getActivityId, activityId),
            eq(PickSourceMapping::getProductId, productId),
            eq(PickSourceMapping::getSourceType, SOURCE_TYPE_NATIVE),
            eq(PickSourceMapping::getStatus, 1)
        );

        if (mappings.size() == 1)
            return resolved(mappings.get(0));
        if (mappings.size() == 0)
            return unresolved(REASON_COLONEL_MAPPING_NOT_FOUND);
        return ambiguous(); // 多个达人操作同一商品
    }
}
```

## 核心业务不变式（Invariant）

### 不变式1：PickSourceMapping NATIVE 唯一性

```
给定的 (colonelBuyinId, activityId, productId, sourceType=NATIVE)
最多只能对应一个 userId

场景：
- 达人A 生成活动3859423商品X的推广链接 → 写入 native mapping(userId=A)
- 达人B 也生成活动3859423商品X的推广链接 → 发现已有 mapping(userId=A)
  → 标记为 AMBIGUOUS，两个人的订单都不归因

实现：PickSourceMappingService.findExistingMapping() 中的
NATIVE 分支按 (colonelBuyinId, activityId, productId, userId) 查找
UPSERT 时发现 existing.userId != newUserId → 拒绝写入
```

### 不变式2：Order Attribution 只读

```
订单归因一旦完成，extraData 不可再修改

场景：
- 订单 3859423 首次归因失败（COLONEL_MAPPING_NOT_FOUND）
- 后续用户转链生成了 native mapping
- 历史订单不会重新归因（除非手动触发 replay）

实现：Order 聚合的 attribution 字段是私有的
只能通过 AttributionService.resolveAttribution() 更新
归因服务本身幂等，重新执行不会破坏数据
```

### 不变式3：ColonelBuyinId 解析一致性

```
同一 activityId + productId 在同一时刻
ColonelBuyinIdResolver 应返回稳定结果

实现：
- 结果带 source 标记，日志可完整追溯来源
- 分层策略按固定顺序执行：Snapshot → OperationState → Activity → API
```
