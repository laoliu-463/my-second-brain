# 七领域设计总览

## 概述

抖音团长 SaaS 的新设计文档把系统拆成 7 个业务域和 1 个分析模块：商品、达人、寄样、订单、业绩、用户、配置负责业务事实和规则，分析模块作为末端消费者维护汇总表，避免看板直接扫业务明细。

## 详细内容

### 顶层边界

| 领域 | 一句话职责 | 核心实体 |
|------|------------|----------|
| 商品域 | 管理团长活动、合作方、商品及活动商品关联 | 活动、合作方、商品、活动商品关联、`pick_source` 映射 |
| 达人域 | 管理达人是谁、归谁跟进、保护期、地址和标签 | 达人、认领记录、独家达人记录 |
| 寄样域 | 管理样品申请、审核、发货、物流和交作业 | 寄样申请、寄样历史 |
| 订单域 | 同步抖音订单事实，保存默认归因和双轨金额 | 订单、订单同步记录、`pick_source` 映射 |
| 业绩域 | 根据默认归因和规则计算最终归属、提成与毛利 | 业绩记录、独家商家记录 |
| 用户域 | 管理用户、角色、组织架构、数据范围与审计 | 用户、角色、部门、组别、操作日志 |
| 配置域 | 管理系统级业务规则和模板参数 | 系统配置、配置历史、差异化提成预留 |
| 分析模块 | 订阅业务事件并维护看板汇总 | 日汇总表、排行榜汇总、查询 API |

### 通用设计规则

| 规则 | 含义 |
|------|------|
| 跨域不写表 | 一个领域不能直接更新另一个领域的数据表 |
| 写副作用走事件 | 订单同步、寄样完成、业绩计算等变化通过事件通知下游 |
| 读聚合走查询接口 | BFF 或内部 Query API 可以同步读取其他领域只读数据 |
| 状态机内聚 | 活动商品、达人认领、寄样申请、订单等生命周期在本领域维护 |
| 配置集中 | 保护期、寄样限制、提成比例、模板规则由配置域统一管理 |
| 分析独立 | 看板读汇总表，分析模块不重新实现业绩归属规则 |

### 核心事件

| 事件 | 发出者 | 订阅者 | V1 用途 |
|------|--------|--------|---------|
| 订单已同步 | 订单域 | 业绩域、寄样域、分析模块 | 触发业绩计算、交作业匹配、看板汇总 |
| 寄样已提交 | 寄样域 | 业绩域 | 记录寄样行为，V2 可用于独家判断 |
| 寄样已通过 | 寄样域 | 业绩域 | 统计有效寄样 |
| 寄样已完成 | 寄样域 | 业绩域、达人域 | 出单完成后更新统计与保护期 |
| 商品已上架 | 商品域 | 前端、寄样域 | 商品库展示与寄样入口可用 |
| 商品负责人变更 | 商品域 | 业绩域 | V1 不做重算，V2 预留 |
| 达人已认领 / 已释放 | 达人域 | 前端、分析模块 | 私海/公海变化与统计 |
| 配置已变更 | 配置域 | 各业务域 | Redis 缓存失效，规则热更新 |
| 业绩已计算 | 业绩域 | 分析模块 | 汇总表更新与看板口径统一 |

### 三条业务链的领域流转

| 业务链 | 领域流转 |
|--------|----------|
| 渠道链 | 达人域认领 -> 商品域选品与转链 -> 寄样域申请 -> 订单域同步订单 -> 业绩域计算 -> 分析模块汇总 |
| 招商链 | 商品域同步活动商品 -> 商品补充与上架 -> 寄样域审核 -> 订单域默认招商归因 -> 业绩域计算 |
| 管理链 | 用户域配置人员和角色 -> 配置域配置规则 -> 各域读取配置并执行 |

### 领域关键规则

| 领域 | 关键规则 |
|------|----------|
| 商品域 | 商品上架前需补充寄样要求、卖点、活动信息；复制讲解生成带 `pick_source` 的推广链接 |
| 达人域 | 多人可同时认领同一达人；保护期内仅认领人可见；出单可重置保护期 |
| 寄样域 | 7 天内同一渠道、达人、商品不能重复申请；组长/管理员可豁免；被拒绝后可重新申请 |
| 订单域 | 只存订单事实、默认归因和双轨金额，不计算提成，不应用独家规则 |
| 业绩域 | V1 最终归属等于默认归属；V2 再启用独家达人/独家商家覆盖 |
| 用户域 | V1 角色、菜单、操作权限和数据范围是各域过滤数据的基础 |
| 配置域 | 提成比例变更不自动重算历史业绩，需要管理员手动触发 |
| 分析模块 | 预估与结算采用双时间口径，提成结果以业绩域为准 |

### V1 与 V2 的边界

| 能力 | V1 | V2 |
|------|----|----|
| 默认归因 | 实现 | 继续保留 |
| 独家达人 | 不启用 | 达人域判定，业绩域应用 |
| 独家商家 | 不启用 | 业绩域判定和应用 |
| 物流自动跟踪 | 口径待确认 | 对接物流供应商 |
| 差异化提成 | 不启用 | 配置域扩展差异化提成表 |
| 标签预设库 | 不启用，开放输入 | 配置域维护预设标签 |
| 看板增强 | 简化趋势与排行 | 增强维度、导出、更多排行 |
