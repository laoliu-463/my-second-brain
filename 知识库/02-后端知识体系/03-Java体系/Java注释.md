# Java 注释

> 核心用途：给代码加说明，注释本身不会被执行。文档注释可生成 API 文档。

---

## 注释类型一览

| 类型 | 语法 | 用途 | 特点 |
|------|------|------|------|
| **单行注释** | `// 内容` | 代码旁简单提示 | 只管一行 |
| **多行注释** | `/* 内容 */` | 多行说明或临时屏蔽代码 | 可跨多行 |
| **文档注释** | `/** 内容 */` | 为类、方法、字段加说明，可生成文档 | 支持 `@标签` |

---

## 文档注释标签

| 标签 | 用途 | 厨师类比 |
|------|------|---------|
| `@author` | 作者 | 谁写的菜谱 |
| `@version` | 版本号 | 菜谱版本 |
| `@since` | 自哪天起引入 | 菜谱开始使用时间 |
| `@param` | 方法参数说明 | 做菜的原料清单 |
| `@return` | 方法返回值 | 做完菜的成果/菜品 |
| `@throws` / `@exception` | 可能抛出的异常 | 做菜可能出现的风险提示 |

---

## 厨师类比记忆

- **单行注释** → 菜谱便签（小提示，一行）
- **多行注释** → 菜谱说明段（详细步骤）
- **文档注释** → 附带说明书的菜谱（可生成文档）
- **标签** → 菜谱元信息（作者、版本、原料、结果、风险）

---

## 面试要点

1. 注释不会被 JVM 执行（编译时忽略）
2. 文档注释支持特殊标签，可被 Javadoc 工具提取生成 API 文档
3. 标签是工具可识别的结构化信息，区别于普通文字说明
