# Java 静态与实例变量及方法

> 重要基础：连接 static 关键字、类和对象的区别、方法区/堆内存、Spring Bean、工具类、单例模式、线程安全。

---

## 一、分类总览

```
类的成员
├── 成员变量
│   ├── 实例变量：没有 static
│   └── 静态变量：有 static
└── 成员方法
    ├── 实例方法：没有 static
    └── 静态方法：有 static
```

**一句话**：没有 static 属于对象，有 static 属于类。

---

## 二、实例变量 vs 静态变量

| 对比点 | 静态变量（static） | 实例变量 |
|--------|------------------|---------|
| 是否有 static | 有 | 无 |
| 属于谁 | 类，所有对象共享一份 | 对象，每对象独立一份 |
| 生命周期 | 随类加载初始化 | 随对象创建存在 |
| 访问方式 | `类名.变量名`（推荐）| `对象名.变量名` |
| 典型场景 | 常量、共享配置、计数器 | 姓名、年龄、订单金额等对象属性 |

---

## 三、实例方法 vs 静态方法

| 对比点 | 静态方法（static） | 实例方法 |
|--------|------------------|---------|
| 是否有 static | 有 | 无 |
| 属于谁 | 类 | 对象 |
| 调用方式 | `类名.方法名()`（推荐）| `对象名.方法名()` |
| 需要创建对象 | 不需要 | 需要 |
| 直接访问实例变量 | **不能** | 能 |
| 直接访问静态变量 | 能 | 能 |
| 常见用途 | 工具方法、工厂方法 | 操作对象自身数据 |

---

## 四、静态方法为什么不能直接访问实例变量

静态方法属于类，执行时可能还没有创建任何对象：

```java
Student.show();  // 类名调用，此时没有对象
```

JVM 不知道要访问哪个对象的 name，所以编译直接报错。

**实例方法**通过对象调用，有对象自然能访问实例变量。

---

## 五、代码示例

```java
public class Student {
    String name;                          // 实例变量
    static String school = "DJTU";        // 静态变量

    public void study() {                 // 实例方法
        System.out.println(name + " 在 " + school + " 学习");
    }

    public static void showSchool() {     // 静态方法
        System.out.println("学校：" + school);
    }
}
```

```java
Student s1 = new Student();
Student s2 = new Student();
s1.name = "张三";
s2.name = "李四";
Student.school = "大连交通大学";

s1.study();         // 张三 在 大连交通大学 学习
s2.study();         // 李四 在 大连交通大学 学习
Student.showSchool(); // 学校：大连交通大学
```

## 六、常见使用场景

### 静态变量适合

- 全局共享信息：`static String school`
- 常量：`static final String PI = "3.14"`
- 计数器：`static int count = 0`

### 静态方法适合

- 工具方法：`MathUtil.add()`
- 工厂方法：`ValueOf()`
- 不依赖对象状态的逻辑

## 七、面试标准回答

**Q：静态变量和实例变量的区别？**

> 静态变量使用 static 修饰，属于类本身，整个类只有一份，被所有对象共享，通常通过类名访问，适合常量、共享配置、计数器。实例变量没有 static 修饰，属于具体对象，每创建一个对象就有一份独立实例变量，随对象创建而存在。

**Q：静态方法和实例方法的区别？**

> 静态方法使用 static 修饰，属于类，可通过类名直接调用，不需要对象。静态方法不能直接访问实例变量和实例方法，因为实例成员属于对象，而静态方法执行时可能还没有对象。实例方法属于对象，需要通过对象调用，可以访问实例成员和静态成员，通常用于操作对象自身数据。
