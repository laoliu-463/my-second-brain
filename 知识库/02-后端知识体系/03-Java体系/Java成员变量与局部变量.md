# Java 成员变量与局部变量

> 重要基础：连接 JVM 内存结构、栈和堆、对象创建、默认值、static、线程安全、变量作用域。

---

## 一、变量分类总览

```
变量
├── 成员变量
│   ├── 实例变量（无 static）
│   └── 静态变量（有 static）
└── 局部变量（方法内、代码块内、方法参数）
```

**一句话**：
> 成员变量：写在类里面、方法外面。局部变量：写在方法里面、代码块里面、方法参数里。

---

## 二、类比理解

- **成员变量** = 房子自带的家具，对象存在它就存在
- **局部变量** = 临时拿出来的工具，方法结束它就消失

---

## 三、成员变量

### 实例变量（无 static）

每创建一个对象，每个对象都有自己的一份。

```java
Student s1 = new Student();
Student s2 = new Student();
s1.name = "张三";
s2.name = "李四";
// s1.name = "张三"，s2.name = "李四"，各一份
```

### 静态变量（有 static）

属于类，不属于某个对象，所有对象共享同一份。

```java
static String school = "大连交通大学";
Student.school;  // 推荐用类名访问
```

---

## 四、局部变量

定义在方法内部、代码块内部、方法参数位置。

```java
public void test(int num) {    // num：方法参数
    int a = 10;                // a：方法内
    if (a > 5) {
        int b = 20;            // b：代码块内
    }
}
```

作用范围：`num` 整个方法可用，`a` 方法内可用，`b` 仅 if 块内可用。

---

## 五、核心区别对比表

| 对比点 | 成员变量 | 局部变量 |
|--------|---------|---------|
| 定义位置 | 类中，方法外 | 方法内、代码块内、参数 |
| 作用范围 | 整个类中可用 | 所在方法或代码块内 |
| 默认值 | **有默认值** | 无默认值，必须先赋值 |
| 生命周期 | 随对象创建/销毁 | 随方法调用创建，方法结束销毁 |
| 存储位置 | 实例变量在堆；静态变量在类相关区域 | 栈帧的局部变量表中 |
| 修饰符 | 可用 `private/public/static` 等 | 不能用访问权限修饰符 |
| 重名规则 | 可与局部变量重名 | 重名时局部变量优先 |

---

## 六、最重要区别1：默认值

### 成员变量有默认值

```java
class Student {
    int age;       // 0
    boolean flag;  // false
    String name;   // null
}
```

### 局部变量无默认值

```java
public void test() {
    int a;
    System.out.println(a);  // 编译报错
}
```

---

## 七、最重要区别2：重名与 this

局部变量和成员变量重名时，局部变量优先：

```java
class Student {
    String name = "成员变量";

    public void test() {
        String name = "局部变量";
        System.out.println(name);        // 局部变量
        System.out.println(this.name);   // 成员变量
    }
}
```

方法参数也是局部变量：

```java
public void setName(String name) {
    this.name = name;  // this.name=成员变量，name=局部变量(参数)
}
```

---

## 八、线程安全浅概念

- **局部变量**：线程安全，每个方法调用有自己独立的局部变量
- **成员变量**（实例变量）：非线程安全，多线程共享同一个对象的实例变量，可能产生竞态条件

---

## 九、面试标准回答

**Q：成员变量和局部变量的区别？**

> 成员变量定义在类中、方法外，属于对象或类的一部分；局部变量定义在方法、代码块或方法参数中，只在当前方法或代码块内有效。
>
> 成员变量有默认值，实例变量随对象创建而创建，随对象销毁而销毁；静态变量属于类，随类加载初始化。局部变量没有默认值，必须先赋值再使用，随方法调用进入栈帧，方法结束后销毁。
>
> 如果成员变量和局部变量重名，局部变量优先；访问当前对象的成员变量可以使用 this。
