# Java 移位运算符

> 一句话记住：移位运算符用于"用二进制位来提高效率、压缩信息、做底层计算"。

---

## 一、基础运算规则

```java
int a = 8;
int b = a << 1;   // 16，左移1位 = 乘以2
int c = a >> 1;   // 4，右移1位 = 除以2
```

| 运算符 | 名称 | 效果 |
|--------|------|------|
| `<<` | 左移 | `a << n` ≈ `a * 2^n` |
| `>>` | 右移（带符号） | `a >> n` ≈ `a / 2^n`，负数高位补1 |
| `>>>` | 无符号右移 | 高位补0，不保留符号 |

> 注意：业务代码里直接写 `num * 2` 比 `num << 1` 更可读，不要为了炫技用移位。

---

## 二、核心应用场景

### 场景1：乘除 2 的幂（集合扩容）

**HashMap 扩容**：默认容量 16，每次扩容翻倍（16→32→64→128）

```java
newCap = oldCap << 1;   // 新容量 = 旧容量 * 2
```

类比：仓库容量不够时直接翻倍，左移就是"快速翻倍"的工具。

### 场景2：权限控制（位标记）

用单个 int 的不同二进制位存储多个权限状态：

```java
int READ   = 1 << 0;    // 0001
int WRITE  = 1 << 1;    // 0010
int DELETE = 1 << 2;    // 0100
int ADMIN  = 1 << 3;    // 1000

// 组合权限
int permission = READ | WRITE;

// 判断是否有写权限
boolean hasWrite = (permission & WRITE) != 0;
```

类比：一个整数像一排开关，每一位代表一个权限，1开0关。

适用：权限系统、状态标记、配置开关、文件权限。

### 场景3：状态压缩

多个布尔状态压到一个整数的不同位：

```java
int ENABLED = 1 << 0;   // 0001
int DELETED = 1 << 1;   // 0010
int CHECKED = 1 << 2;   // 0100

int status = ENABLED | CHECKED;  // 启用+审核
```

优点：节省空间、判断速度快。缺点：可读性差，业务代码慎用。

### 场景4：哈希扰动（HashMap 源码）

```java
hash ^ (hash >>> 16)
```

`>>>` 无符号右移 16 位，让高位信息参与低位计算，减少哈希冲突。

类比：像把饮料上下搅拌，让成分分布均匀，避免糖沉底。

### 场景5：网络协议 / 文件格式解析

把多个字节拼成一个整数：

```java
int value = (b1 << 24) | (b2 << 16) | (b3 << 8) | b4;
```

常见于：网络协议解析、图片/音视频格式解析、加密算法、二进制文件读写。

---

## 三、面试标准回答

**Q：移位运算符一般用在什么场景？**

> 移位运算符主要用于二进制层面的高效计算。常见场景包括乘除 2 的幂、集合源码里的容量扩容（如 HashMap 扩容翻倍）；也用于权限位标记、状态压缩，把多个布尔状态压到一个整数的不同位中。另外在哈希扰动、网络协议解析、文件格式解析等底层场景里，也经常用移位来拼接或拆分二进制数据。
>
> 普通业务代码里不会为了炫技用移位，可读性不好。但在源码、底层协议、权限位标记这些场景里，移位很有价值。

---

## 四、初试重点记三个场景

1. **乘除 2 的幂**：HashMap 扩容翻倍
2. **HashMap 哈希扰动**：`hash ^ (hash >>> 16)`
3. **权限控制 / 状态标记**：位运算组合判断权限
